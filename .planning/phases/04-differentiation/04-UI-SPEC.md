# Phase 4 — UI Design Contract (Terminal)

> Visual and interaction contract for Phase 4: Differentiation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This project is a CLI application rendered in terminal via React + Ink. All values are in terminal character cells, not pixels. Typography is monospace; color is ANSI.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (terminal CLI — no web design system) |
| Preset | not applicable |
| Component library | ink 7.0.1 + @inkjs/ui 2.0.0 |
| Icon library | Unicode symbols + figlet (ASCII art headers) |
| Font | Terminal monospace (user's terminal font) |
| CJK support | string-width 8.2.0 for display width calculation |

---

## Spacing Scale (Character Cells)

All spacing measured in terminal character cells (1 cell = 1 monospace character width).

| Token | Horizontal (chars) | Vertical (lines) | Usage |
|-------|--------------------:|------------------:|-------|
| xs | 1 | 0 | Inline gaps between symbols, single-char padding |
| sm | 2 | 1 | Compact element spacing, entry margins |
| md | 4 | 1 | Standard content indentation (`paddingX={2}` in Ink) |
| lg | 6 | 2 | Section gaps, panel internal padding |

Ink mapping:
- `paddingX={1}` = 1 char each side (established pattern, keep for all panels)
- `marginTop={1}` / `marginBottom={1}` = 1 line vertical spacing (established)
- Indentation: 2 chars for nested content (e.g., `  ` prefix for tree children, codex details)
- Deep nesting: 4 chars for second-level children (e.g., branch tree grandchildren)

Exceptions: none

## Typography (Terminal Text Styling)

Terminal monospace has no font size or weight control. Style via Ink `<Text>` props:

| Role | Ink Props | Usage |
|------|-----------|-------|
| Body | `<Text>` (no props) | Narration text, diff content, codex descriptions, map labels |
| Label | `<Text dimColor>` | Hints, keyboard shortcuts, metadata, inactive items, secondary info |
| Heading | `<Text bold color="cyan">` | Panel titles: `【分支树】`, `【地图】`, `【知识典籍】`, `【快捷键】` |
| Subheading | `<Text bold color="yellow">` | Section headers within panels: category names, status groups |
| Emphasis | `<Text bold>` | Selected items, important values, node names |
| Highlight | `<Text color="cyan">` | Current selection cursor, active branch, current location |

Source: Established pattern from Phase 1-3 codebase (title-bar.tsx, journal-panel.tsx, actions-panel.tsx).

## Panel Layout Contracts

### 1. Branch Tree Panel (`/branch tree`)

**Trigger:** `/branch tree` command or `b` shortcut (non-input mode)
**Replaces:** Scene panel area (same slot as JournalPanel)

```
【分支树】                                      Esc 返回

  main ──● save-01 (Day 1, 黑松镇)
    │    └── save-02 (Day 2, 幽暗森林)
    │
    ├── rescue ──● save-03 (Day 2, 矿洞)    ← current
    │            └── save-04 (Day 3, 黑松镇)
    │
    └── betray ──● save-05 (Day 2, 暗道)

↑↓ 选择节点    Enter 查看详情    c 对比    Esc 返回
```

**Layout contract:**
- Render inside scene panel slot (replace ScenePanel content)
- `paddingX={1}` consistent with other panels
- Header: `<Text bold color="cyan">【分支树】</Text>` left, `<Text dimColor>Esc 返回</Text>` right
- Tree lines: `│`, `├──`, `└──` using Unicode box-drawing characters
- Current branch: `← current` suffix in cyan
- HEAD save: `●` marker (filled circle)
- Other saves: `○` marker (empty circle)
- Each node: `{save-name} ({game-time}, {location})` in body text
- Selected node: `bold` + `color="cyan"`
- Scrollable if tree exceeds panel height (Ink scrollable Box or manual viewport)
- **Wide mode (>= 100):** Tree left 60%, detail pane right 40% showing expanded info for selected node
- **Narrow mode (< 100):** Full-width tree, Enter opens detail as overlay

**Detail view fields:**
- Save name, game time, current location
- Key quest stages (brief, 1-2 lines)
- Key NPC relationship changes (trust/hostility delta, max 3 NPCs)
- Branch source: "从 {parent-branch} 的 {save-name} 分出"

### 2. Compare View (`/compare`)

**Trigger:** `/compare` command or `/compare main..rescue`
**Replaces:** Scene panel area

```
【分支对比】main ↔ rescue                       Esc 返回

  摘要: 3 项差异，1 项高影响分歧

  ── 任务进度 ──
  ~ 狼灾调查: 阶段 2 → 阶段 3
  + 矿洞探索 (rescue 独有)

  ── NPC 关系 ──
  ~ 守卫队长·陈刚: 信任 +15 → +25                       !高影响
  - 商人·老赵: 关系 -5 (main 仅有)

  ── 背包 ──
  + 矿洞钥匙 (rescue 独有)

  ── 叙事影响 ──
  rescue 分支中，你选择救援矿工，获得了守卫队的信任...

↑↓ 滚动    Tab 切换并排/列表    Esc 返回
```

**Layout contract:**
- Header: `<Text bold color="cyan">【分支对比】</Text>` + branch names in body text
- Summary line: count of diffs + count of high-impact divergences
- Category headers: `── {category} ──` using em-dash, `<Text bold dimColor>`
- Diff markers at line start:
  - `+` = `<Text color="green">+</Text>` (addition)
  - `-` = `<Text color="red">-</Text>` (removal)
  - `~` = `<Text color="yellow">~</Text>` (change)
  - `!` = `<Text color="yellow" bold>!</Text>` (high-impact divergence, appended at line end)
- Diff content: body text (no color) after marker
- Narrative summary at bottom: 1-2 lines in body text, summarizing branch divergence meaning
- **Wide mode (>= 100):** Optional side-by-side columns (`Tab` toggle), left = branch A, right = branch B, center divider `│`
- **Narrow mode (< 100):** Single column unified diff only
- Scrollable for long diffs

**Compare dimensions (in order):**
1. Quest progress
2. NPC relationships / reputation
3. Inventory
4. Location
5. Faction reputation
6. Discovered truths / player knowledge

### 3. ASCII Map Panel (`/map`)

**Trigger:** `/map` command or `m` shortcut (non-input mode)
**Replaces:** Scene panel area

```
【地图】铁风谷地区                               Esc 返回

        [?]
         │
  [F]───[H]───[T]
   │     │
  [?]   [D]
         │
        [M]  ← 当前位置

  图例: [H]城镇 [T]神殿 [F]森林 [D]地牢 [M]矿洞 [?]未知

  ── 黑松镇 ──
  类型: 城镇  危险: 低  状态: 已勘察
  出口: 北→幽暗森林  东→铁风神殿  南→矿道入口

↑↓←→ 移动选择    Enter 查看详情    Esc 返回
```

**Layout contract:**
- Header: `<Text bold color="cyan">【地图】</Text>` + region name in body text
- Map grid: fixed-width character grid, centered in panel
- Path lines: `─` horizontal, `│` vertical, `┼` crossing (body text, no color)
- Node format: `[X]` where X is single-char icon

**Node icons and exploration states:**

| Icon | Meaning | Color |
|------|---------|-------|
| `[H]` | Town (城镇) | body text when visited, `dimColor` when known |
| `[T]` | Temple (神殿) | body text when visited, `dimColor` when known |
| `[F]` | Forest (森林) | body text when visited, `dimColor` when known |
| `[D]` | Dungeon (地牢) | body text when visited, `dimColor` when known |
| `[M]` | Mine (矿洞) | body text when visited, `dimColor` when known |
| `[?]` | Unknown | `dimColor` always |
| `[!]` | Quest-related | `color="magenta"` |

**Exploration state rendering:**

| State | Node Style | Path Style |
|-------|------------|------------|
| `unknown` | Not rendered (hidden) |  |
| `rumored` | `<Text dimColor>[?]</Text>` | `<Text dimColor>---</Text>` dashed |
| `known` | `<Text dimColor>[X]</Text>` | `<Text dimColor>───</Text>` |
| `visited` | `<Text>[X]</Text>` | `<Text>───</Text>` |
| `surveyed` | `<Text bold>[X]</Text>` | `<Text>───</Text>` |

- Current location: `← 当前位置` suffix in `<Text color="cyan">`
- Legend bar: below map, single line, `dimColor`
- Detail panel: below legend, shows selected location info
  - Location name as `<Text bold>` subheading
  - Type, danger level, status on one line
  - Exits listed with directions and destination names
- **Wide mode (>= 100):** Map left 50%, detail pane right 50%
- **Narrow mode (< 100):** Map top, detail bottom (stacked)

### 4. Codex Browser Panel (`/codex`)

**Trigger:** `/codex` command or `c` shortcut (non-input mode)
**Replaces:** Scene panel area

```
【知识典籍】                          搜索: ▏       Esc 返回

  种族(3)  阵营(4)  地点(9)  法术(2)  物品(5)  历史(3)

  ❯ 黑松镇                                    地点 · 确证
    铁风谷                                     地点 · 确证
    幽暗森林                                   地点 · 传闻
    ???                                        地点 · ???

  ── 黑松镇 ──
  类型: 地点  权威: regional_common_knowledge
  可信度: 0.9  来源: 探索

  铁风谷东部的边陲小镇，以木材贸易闻名。
  近来频繁的狼灾困扰着镇上居民...

  关联: 守卫队长·陈刚, 商人·老赵, 铁风守卫团

↑↓ 选择    Tab 搜索    Enter 查看    / 过滤    Esc 返回
```

**Layout contract:**
- Header: `<Text bold color="cyan">【知识典籍】</Text>` left, search input right (inline TextInput from @inkjs/ui)
- Category tabs: horizontal list with counts, `<Text bold color="yellow">` for active category, `<Text dimColor>` for inactive
- Entry list: vertical, `❯` cursor for selected item
  - Entry name: `<Text bold>` if selected, `<Text>` otherwise
  - Right-aligned: `<Text dimColor>{type} . {knowledge_status}</Text>`
  - Hidden entries: `<Text dimColor>???</Text>` (name replaced with `???`, metadata replaced with `???`)
- Detail pane (below list or right side):
  - Entry name as `<Text bold>` subheading
  - Metadata line: type, authority, confidence, source -- all `dimColor`
  - Description body: plain text, no color
  - Related entries: `<Text color="blue">` for clickable related entity names

**Visibility rendering:**

| Visibility | Entry Name | Content |
|------------|-----------|---------|
| `public` | Full name | Full description |
| `discovered` | Full name | Full description |
| `hidden` | `???` | `<Text dimColor>尚未发现相关信息。继续探索以揭示更多知识。</Text>` |
| `secret` | `???` | `<Text dimColor>某些秘密仍然隐藏在迷雾之中...</Text>` |
| `forbidden` | Not shown | Not shown (filtered from list entirely) |

**Knowledge status badges:**

| Status | Display | Color |
|--------|---------|-------|
| `heard` | 传闻 | `<Text color="blue">` |
| `suspected` | 推断 | `<Text color="blue">` |
| `confirmed` | 确证 | `<Text color="green">` |
| `contradicted` | 存疑 | `<Text color="red">` |

- **Wide mode (>= 100):** Two-column: left 40% = search + category tabs + entry list, right 60% = detail pane
- **Narrow mode (< 100):** Single column, Enter on entry opens detail view replacing list, Esc returns to list
- Search: fuzzy match on entry name and tags (Chinese text + pinyin if feasible)
- Filter by category via Tab cycling or `/` type filter prefix

### 5. Shortcut Help Panel (`?` or `/help shortcuts`)

**Trigger:** `?` key (non-input mode) or `/help shortcuts`
**Replaces:** Scene panel area (overlay)

```
【快捷键】                                      Esc 返回

  ── 核心操作 ──
  Tab          命令/NPC名/方向补全
  ↑ ↓          浏览历史 / 列表选择
  Enter        确认选择
  Esc          取消 / 返回上级
  /            切换至命令输入
  ?            显示此帮助

  ── 面板切换（非输入状态）──
  m            打开地图        = /map
  j            打开任务日志    = /journal
  c            打开知识典籍    = /codex
  i            打开背包        = /inventory
  b            打开分支树      = /branch tree

  ── 地图/列表导航 ──
  ↑ ↓ ← →     地图移动 / 列表滚动
  Enter        查看详情
  Ctrl-R       搜索历史

Esc 关闭帮助
```

**Layout contract:**
- Header: `<Text bold color="cyan">【快捷键】</Text>`
- Category headers: `── {name} ──` in `<Text bold dimColor>`
- Key column: `<Text bold color="cyan">{key}</Text>` left-aligned, fixed 14-char width
- Description column: `<Text>{description}</Text>`
- Command equivalent: `<Text dimColor>= /{command}</Text>` right-aligned (for panel shortcuts only)
- Single column layout, scrollable if needed
- No wide/narrow difference (same layout for all widths)

## CJK Text Handling Contract

| Rule | Implementation |
|------|---------------|
| All display width calculations | Use `stringWidth()` from `string-width` package, NEVER `string.length` |
| Truncation | Iterate character-by-character with `stringWidth()` per char, stop when accumulated width + ellipsis width exceeds max (established pattern in status-bar.tsx) |
| Map node alignment | Each `[X]` node occupies 3 display columns; pad with spaces to align grid |
| Codex entry list | Right-align metadata badges using `stringWidth()` of entry name to calculate remaining space |
| Branch tree alignment | Tree lines use fixed-width prefixes; save labels may contain CJK, calculate with `stringWidth()` |
| Diff content | Marker column is 2 chars fixed (`+ `, `- `, `~ `), content follows with natural CJK wrapping |
| Panel headers | `【】` brackets are 2 display columns each (full-width); account for this in header width calculations |
| Category tabs | Calculate tab widths with `stringWidth()` for CJK category names + count suffix |
