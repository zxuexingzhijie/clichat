Add `SpatialExitSchema` and extend `LocationSchema` for backward-compatible spatial data (per D-05):

1. Add `SpatialExitSchema` before LocationSchema:
```typescript
export const SpatialExitSchema = z.object({
  direction: z.string(),
  targetId: z.string(),
  distance: z.number().optional(),
  label: z.string().optional(),
});
export type SpatialExit = z.infer<typeof SpatialExitSchema>;
```

2. Modify `LocationSchema.exits` from `z.array(z.string())` to:
```typescript
exits: z.array(z.union([z.string(), SpatialExitSchema])),
```
This is backward-compatible: old `["loc_main_street"]` still validates, and new `[{ direction: "north", targetId: "loc_main_street" }]` also validates.

3. Add optional fields to LocationSchema:
```typescript
coordinates: z.object({ x: z.number(), y: z.number() }).optional(),
map_icon: z.string().optional(),
```

4. Export `SpatialExit` type alongside other exports at the bottom of the file.

Do NOT change any other schema in the file. The `CodexEntrySchema` discriminated union automatically picks up LocationSchema changes since it references the same variable.
