Create `homebrew/` directory as the scaffold for the separate `homebrew-chronicle` repository. This directory will be copied to the separate repo by the user.

Create `homebrew/Formula/chronicle.rb`:
```ruby
class Chronicle < Formula
  desc "AI-driven CLI interactive novel game"
  homepage "https://github.com/OWNER/chronicle-cli"
  version "1.1.0"
  license "MIT"

  on_macos do
    if Hardware::CPU.arm?
      url "https://github.com/OWNER/chronicle-cli/releases/download/v#{version}/chronicle-darwin-arm64.tar.gz"
      sha256 "PLACEHOLDER_DARWIN_ARM64"
    else
      url "https://github.com/OWNER/chronicle-cli/releases/download/v#{version}/chronicle-darwin-x64.tar.gz"
      sha256 "PLACEHOLDER_DARWIN_X64"
    end
  end

  on_linux do
    url "https://github.com/OWNER/chronicle-cli/releases/download/v#{version}/chronicle-linux-x64.tar.gz"
    sha256 "PLACEHOLDER_LINUX_X64"
  end

  def install
    bin.install "chronicle"
  end

  test do
    assert_match version.to_s, shell_output("#{bin}/chronicle --version")
  end
end
```

Notes on PLACEHOLDER values:
- `PLACEHOLDER_DARWIN_ARM64`, `PLACEHOLDER_DARWIN_X64`, `PLACEHOLDER_LINUX_X64` are replaced by the CI release workflow's `update-homebrew` job after building binaries and computing SHA256 hashes
- `OWNER` is the GitHub username/org -- user replaces this
- The formula uses `on_macos` + `on_linux` blocks with `Hardware::CPU.arm?` for platform detection per D-09
- `bin.install "chronicle"` installs the standalone binary to Homebrew's bin directory per D-08

Also add a `.github/workflows/update-formula.yml` to the homebrew scaffold for receiving the `repository_dispatch` from the main repo's release workflow:

Create `homebrew/.github/workflows/update-formula.yml`:
```yaml
name: Update Formula

on:
  repository_dispatch:
    types: [update-formula]

permissions:
  contents: write

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Update formula version and hashes
        run: |
          VERSION="${{ github.event.client_payload.version }}"
          VERSION="${VERSION#v}"  # Strip leading 'v'

          sed -i \
            -e "s/version \".*\"/version \"${VERSION}\"/" \
            -e "s/PLACEHOLDER_DARWIN_ARM64/${{ github.event.client_payload.darwin_arm64_sha256 }}/" \
            -e "s/PLACEHOLDER_DARWIN_X64/${{ github.event.client_payload.darwin_x64_sha256 }}/" \
            -e "s/PLACEHOLDER_LINUX_X64/${{ github.event.client_payload.linux_x64_sha256 }}/" \
            Formula/chronicle.rb

          # Also update any existing SHA256 hashes from previous releases
          sed -i \
            -e "/darwin-arm64/{ n; s/sha256 \"[a-f0-9]*\"/sha256 \"${{ github.event.client_payload.darwin_arm64_sha256 }}\"/; }" \
            -e "/darwin-x64/{ n; s/sha256 \"[a-f0-9]*\"/sha256 \"${{ github.event.client_payload.darwin_x64_sha256 }}\"/; }" \
            -e "/linux-x64/{ n; s/sha256 \"[a-f0-9]*\"/sha256 \"${{ github.event.client_payload.linux_x64_sha256 }}\"/; }" \
            Formula/chronicle.rb

      - name: Commit and push
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add Formula/chronicle.rb
          git commit -m "Update chronicle to ${VERSION}"
          git push
```

This completes the Homebrew distribution chain: main repo's release.yml dispatches to this workflow, which auto-updates the formula with new version + SHA256 hashes.
