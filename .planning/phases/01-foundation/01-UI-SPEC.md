# Phase 1 — UI Design Contract

> Visual and interaction contract for the terminal CLI game layout. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Target environment:** Terminal emulator (80x24 minimum). React + Ink 7 + fullscreen-ink.
> **NOT a web app.** All values are in character cells, ANSI color names, and Ink style props.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (terminal app, no shadcn) |
| Preset | not applicable |
| Component library | Ink 7 (Box, Text) + @inkjs/ui (TextInput, Select) |
| Icon library | Unicode symbols only (no icon font) — `>` cursor, `[x]` checks, box-drawing chars |
| Font | Terminal default monospace. CJK rendering depends on terminal font config |
| Title font | figlet with "ANSI Shadow" font + gradient-string cyan-to-magenta |
| Color utility | chalk 5 for non-React string formatting; Ink `<Text color>` for components |

---

## Spacing Scale

All values in character cells (1 cell = 1 monospace character width).

| Token | Value | Usage |
|-------|-------|-------|
| none | 0 | Divider rows (zero padding — `├───┤` fills full width) |
| xs | 1 | Inline gaps between status bar fields (`HP 30/30  MP 8/8`) — 2 spaces |
| sm | 1 | `paddingX={1}` on content rows inside the outer border |
| md | 2 | `paddingX={2}` for scene panel text indentation (reserved, not Phase 1) |
| lg | 1 row | Blank line between narration paragraphs in scene panel |

**Vertical spacing:**
- Title bar: 1 row
- Dividers: 1 row each (4 dividers total)
- Status bar: 1 row
- Input area: 1 row
- Scene panel: flexGrow=1 (fills remaining vertical space)
- Actions panel: 3-7 rows (3-5 action items + 1 header + 1 hint row)

**Exceptions:**
- Status bar field separator is 2 spaces (not a fixed token — fits as many fields as terminal width allows)
- Actions panel items have `  ` (2-space) indent for unselected, `> ` prefix for selected

## Typography

Terminal renders all text as monospace at the terminal's configured font size. "Typography" here means ANSI text attributes applied via Ink `<Text>` props.

| Role | Ink Props | Usage |
|------|-----------|-------|
| Body | `<Text>` (no props) | Scene narration, action item labels, status bar values |
| Heading | `<Text bold>` | Panel headers ("Actions"), title bar game name |
| Secondary | `<Text dimColor>` | Keyboard hints, time/day display, deselected actions |
| Emphasis | `<Text bold color="cyan">` | Active cursor item, important game events |
| System message | `<Text italic dimColor>` | System feedback, loading states |
| Dice result | `<Text bold>` with inline color spans | Check display: `[D20: 14]` in cyan, result in green/red |

**CJK considerations:**
- All width calculations MUST use `string-width` (not `String.length`)
- Chinese characters occupy 2 terminal columns each
- Status bar truncation logic must account for CJK double-width

## Color

ANSI color names used via Ink `<Text color="name">` and chalk. Auto-detect terminal background (D-10) with dark theme fallback.

### Dark Theme (default)

| Role | ANSI Color | Ink Prop | Usage |
|------|------------|----------|-------|
| Dominant (60%) | terminal default bg | (none — inherit) | Screen background, scene panel, input area |
| Secondary (30%) | dim white | `dimColor` | Status bar labels, keyboard hints, time display, border chars |
| Accent (10%) | cyan | `color="cyan"` | **Reserved for:** selected action cursor, command echo, title bar game name, active input prompt `>`, title screen gradient start |
| Success | green | `color="green"` | Check success results, HP recovery, quest completion |
| Failure | red | `color="red"` | Check failure results, damage taken, HP critical warning |
| Warning | yellow | `color="yellow"` | Low resource alerts (HP < 25%), ambiguous intent, clarification prompts |
| Destructive | red + bold | `color="red" bold` | Death, game over, permanent loss events |

### Light Theme (auto-detected)

| Role | ANSI Color | Notes |
|------|------------|-------|
| Dominant | terminal default bg (light) | Inherit |
| Secondary | dim (dark gray on light bg) | Same `dimColor` prop |
| Accent | blue | Cyan is low-contrast on white bg; use blue instead |
| Success/Failure/Warning | Same as dark | Green/red/yellow work on both backgrounds |

**Accent reserved for (explicit list):**
1. Selected action item cursor (`> 2. 向守卫询问...`)
2. Title bar game name ("Chronicle CLI")
3. Active input prompt character (`>`)
4. figlet title screen gradient (start color)
5. Player command echo in scene panel
6. Active/selected UI element highlight

**Accent is NOT used for:** borders, dividers, status bar values, narration text, NPC dialogue, keyboard hints.

## Border & Divider Contract

Single outer border wrapping entire screen (D-02). Internal horizontal dividers connect to outer border.

| Element | Characters | Rendering Method |
|---------|------------|------------------|
| Outer border | `┌─┐│ │└─┘` | Ink `Box` with `borderStyle="single"` |
| Horizontal divider | `├` + `─`.repeat(width-2) + `┤` | Custom `<Divider>` component as `<Text>` row |
| Title bar left | ` Chronicle CLI ` | `<Text bold color="cyan">` inside `<Box justifyContent="space-between" paddingX={1}>` |
| Title bar right | ` Day 1 / Night ` | `<Text dimColor>` |

**Divider rendering note:** The outer `Box` with `borderStyle="single"` renders its own left/right `│` borders. The `<Divider>` component inside the box renders `├───┤` at the full inner content width. Whether `├` and `┤` visually merge with the outer `│` depends on Ink's border rendering — this MUST be prototyped in Wave 0 (see RESEARCH.md Open Question 1). If misaligned, fall back to rendering the entire border manually as `Text` rows.

| Panel | Rows | Notes |
|-------|------|-------|

| Divider | 1 | Fixed |

## Status Bar Contract

### Field Order (left to right)

`HP {cur}/{max}  MP {cur}/{max}  Gold {n}  Location: {name}  Quest: {name|None}`

### Progressive Collapse (overflow behavior)

When terminal width is insufficient to display all fields on one line:

| Priority | Field | Collapse Behavior |
|----------|-------|-------------------|
| 1 (hides first) | Quest | Hide entirely when width < 65 |
| 2 | Location | Truncate with `...` when width < 55 (e.g., `黑松镇·北...`) |
| 3 | Gold | Hide when width < 45 |
| 4 (always visible) | HP, MP | Always displayed, never hidden |

### Field Styling

| Field | Style | Example |
|-------|-------|---------|
| HP (healthy) | `<Text>` default | `HP 30/30` |
| HP (low, < 25%) | `<Text color="yellow">` | `HP 7/30` |
| HP (critical, < 10%) | `<Text color="red" bold>` | `HP 2/30` |
| MP | `<Text>` default | `MP 8/8` |
| Gold | `<Text>` default | `Gold 12` |
| Location | `<Text>` default | `Location: 黑松镇·北门` |
| Quest | `<Text dimColor>` when None | `Quest: None` |
| Quest | `<Text>` default when active | `Quest: 调查失踪事件` |

## Actions Panel Contract

### Layout

```
Actions                                    ← header, <Text bold>
  1. 仔细阅读告示                           ← unselected, dimColor
❯ 2. 向守卫询问最近的失踪事件               ← selected, bold cyan
  3. 绕到城墙阴影处观察                     ← unselected, dimColor
  4. 打开地图                              ← unselected, dimColor
↑↓ 选择    Enter 确认    / 输入自定义行动   ← hint row, dimColor
```

### Interaction

| Input | Behavior |
|-------|----------|
| `↑` / `↓` | Move cursor between action items |
| `Enter` | Execute selected action |
| `/` | Switch input focus to input area for command mode |
| `1`-`9` | Direct select action by number |
| Any other key | Switch to input area for natural language input |

### Styling

| State | Prefix | Text Style |
|-------|--------|------------|
| Unselected | `  ` (2 spaces) | `<Text dimColor>` |
| Selected | `❯ ` | `<Text bold color="cyan">` |
| Hint row | (none) | `<Text dimColor>` |

### Action Count

- Minimum: 3 actions per scene
- Maximum: 5 actions per scene (D-08)
- Phase 1: placeholder actions hardcoded per scene

## Input Area Contract

### Prompt

| Mode | Prompt Display | Description |
|------|----------------|-------------|
| Action selection (default) | No prompt visible — cursor in actions panel | Arrow keys navigate actions |
| Natural language | `> ` in cyan | Free-form text input, sent to NL intent classifier |
| Command | `/ ` in cyan | After typing `/`, shows command being typed |

### Input Routing (D-37, D-39)

| Input Pattern | Route |
|---------------|-------|
| Starts with `/` | Command parser (Commander.js) |
| Everything else | NL intent classifier (AI SDK v5 generateObject) |

### Tab Completion (D-40)

Available in command mode (`/` prefix):
- Command names: `/look`, `/go`, `/talk`, `/attack`, `/help`, etc.
- Parameters: NPC names, directions, item names (from current scene context)
- Display: inline completion hint in dimColor after cursor position

## Interaction State Machine

```
                    ┌──────────────┐
     app launch ──▶ │ TITLE_SCREEN │
                    └──────┬───────┘
                     any key
                    ┌──────▼───────┐
              ┌────▶│ACTION_SELECT │◀───── default mode
              │     └──────┬───────┘
              │      ↑↓    │ Enter    / or text key
              │     nav    │          │
              │     ┌──────▼──┐  ┌───▼─────────┐
              │     │EXECUTING │  │ INPUT_ACTIVE │
              │     └──────┬──┘  └───┬─────────┘
              │      done  │   Enter │
              │     ┌──────▼─────────▼──┐
              │     │  PROCESSING       │ ← parse command or classify intent
              │     └──────┬────────────┘
              │      result│
              │     ┌──────▼───────┐
              │     │ RULES_ENGINE │ ← adjudicate action
              │     └──────┬───────┘
              │      done  │
              │     ┌──────▼───────┐
              └─────│ SCENE_UPDATE │ ← update panels, new actions generated
                    └──────────────┘
```

### Focus Management

| State | Keyboard captured by | Actions panel | Input area |
|-------|---------------------|---------------|------------|
| ACTION_SELECT | Actions panel (useInput) | Active, cursor visible | Inactive, prompt hidden |
| INPUT_ACTIVE | Input area (TextInput) | Inactive, dimmed | Active, prompt + cursor visible |
| PROCESSING | Neither (show spinner) | Frozen | Frozen |

**Focus switch triggers:**
- ACTION_SELECT → INPUT_ACTIVE: User types `/` or any non-arrow key
- INPUT_ACTIVE → ACTION_SELECT: User submits input (Enter) or presses Escape
- `useInput` must use `{ isActive }` flag to prevent keystroke conflicts (RESEARCH.md Pitfall 5)

## Resize Behavior

| Event | Response |
|-------|----------|
| Terminal resize detected | `useScreenSize` fires, all components re-render with new dimensions |
| Width drops below 80 | Show `SizeGuard` warning, hide game layout |
| Height drops below 24 | Show `SizeGuard` warning, hide game layout |
| Width crosses 100 threshold | Switch between narrow (vertical) and wide (side-by-side) layout |
| Status bar overflow | Progressive collapse: Quest hides first, Location truncates, Gold hides |
| Scene text reflow | Text re-wraps to new width (Ink handles this natively) |
| Divider width | Recalculated from `useScreenSize().width` minus border width (2 chars) |

## Accessibility Notes

Phase 1 baseline (full accessibility is v2 PLAT-04):

| Concern | Phase 1 Approach |
|---------|-----------------|
| Color blindness | Status uses position + label, not color alone. HP/MP always show numbers, not just color |
| Screen reader | Ink renders to stdout — screen readers can parse terminal output. No special ARIA needed |
| High contrast | Auto-detect dark/light theme (D-10). Bold text for emphasis, not color alone |
| Keyboard-only | Entire UI is keyboard-driven by design. No mouse interaction |
