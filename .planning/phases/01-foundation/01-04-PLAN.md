Create `src/ui/components/size-guard.tsx` per UI-SPEC Minimum Terminal Size:
- Props: `{ minWidth: number, minHeight: number, children: React.ReactNode }`
- Uses `useScreenSize()` from 'fullscreen-ink' (or `useTerminalSize` from ink — check which is available)
- If width < minWidth: render `<Box justifyContent="center" alignItems="center"><Text color="yellow">终端窗口过小，请调整至 {minWidth}x{minHeight} 以上</Text></Box>`
- If height < minHeight: same warning
- Otherwise: render children
- Default minimums: 80 columns, 24 rows

Create `src/ui/panels/scene-panel.tsx` per UI-SPEC:
- Props: `{ lines: string[] }`
- Layout: `<Box flexDirection="column" flexGrow={1} paddingX={1}>`
- Each line: `<Text>{line}</Text>`
- Empty state: `<Text dimColor>周围一片寂静，什么也没有发生。</Text>`

Create `src/ui/panels/status-bar.tsx` per UI-SPEC Status Bar Contract (D-05):
- Props: `{ hp: number, maxHp: number, mp: number, maxMp: number, gold: number, location: string, quest: string | null, width: number }`
- Field order: `HP {cur}/{max}  MP {cur}/{max}  Gold {n}  Location: {name}  Quest: {name|None}`
- Progressive collapse per UI-SPEC:
  - width < 65: hide Quest
  - width < 55: truncate Location with `...` (use `string-width` for CJK)
  - width < 45: hide Gold
  - HP/MP always visible
- HP styling: default for healthy, `color="yellow"` for < 25%, `color="red" bold` for < 10%
- Quest: `dimColor` when "None", default when active
- Import `stringWidth` from 'string-width' for CJK-safe truncation

Create `src/ui/panels/actions-panel.tsx` per UI-SPEC Actions Panel Contract (D-06):
- Props: `{ actions: { id: string, label: string }[], selectedIndex: number, onSelect: (index: number) => void, onExecute: (index: number) => void, isActive: boolean }`
- Header: `<Text bold>可选行动</Text>`
- Each action: numbered `{i+1}. {label}`
  - Selected (index === selectedIndex): prefix `❯ `, `<Text bold color="cyan">`
  - Unselected: prefix `  ` (2 spaces), `<Text dimColor>`
- Hint row: `<Text dimColor>↑↓ 选择    Enter 确认    / 输入自定义行动</Text>`
- Use `useInput` from 'ink' with `{ isActive }` to handle:
  - Up arrow: decrement selectedIndex (wrap to last)
  - Down arrow: increment selectedIndex (wrap to first)
  - Enter: call onExecute(selectedIndex)
  - Number keys 1-9: direct select if index < actions.length
- CRITICAL: `isActive` prop must be passed to `useInput` to prevent keystroke conflicts with InputArea (Pitfall 5)

Create `src/ui/panels/input-area.tsx` per UI-SPEC Input Area Contract (D-07):
- Props: `{ onSubmit: (text: string) => void, isActive: boolean, mode: 'action' | 'nl' | 'command' }`
- When mode='action': render nothing (focus is on actions panel)
- When mode='nl': render `<Text color="cyan">{'> '}</Text>` + TextInput from @inkjs/ui
- When mode='command': render `<Text color="cyan">{'/ '}</Text>` + TextInput
- TextInput props: `{ onSubmit: (value) => onSubmit(value), isDisabled: !isActive }`
- On submit: clear input, call onSubmit with the entered text

Create `src/ui/hooks/use-game-input.ts` per UI-SPEC Interaction State Machine:
- Manages the interaction state: 'action_select' | 'input_active' | 'processing'
- Exports: `{ inputMode, setInputMode, selectedActionIndex, setSelectedActionIndex }`
- State machine transitions:
  - action_select: Arrow keys active on ActionsPanel, InputArea inactive
  - input_active: ActionsPanel dimmed, InputArea active with TextInput focus
  - processing: Both frozen (for future AI call integration)
- Focus switch triggers:
  - action_select -> input_active: user types `/` or any non-arrow key (detected in ActionsPanel)
  - input_active -> action_select: user submits input (Enter) or presses Escape
- Track `isTyping` boolean that ActionsPanel uses for `isActive={!isTyping}` and InputArea uses for `isActive={isTyping}`
