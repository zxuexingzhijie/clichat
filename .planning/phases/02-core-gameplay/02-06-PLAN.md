Create `src/engine/dialogue-manager.ts`:
- Factory pattern: `createDialogueManager(codexEntries, options?: { generateNpcDialogueFn?, adjudicateFn? })`
- Dependency injection for AI and adjudication functions

`startDialogue(npcId: string): Promise<DialogueResult>`:
1. Find NPC in codex via `queryById`
2. Get NPC memories from `npcMemoryStore.getState().memories[npcId] || []`
3. Get current scene from `sceneStore.getState()`
4. Call `generateNpcDialogue(npcProfile, scene, 'greet', memories)`
5. Determine mode: if NPC has quest-related goals (goals contains words like 'investigate', 'protect', 'find', 'recruit') OR initial_disposition < -0.2 OR initial_disposition > 0.5, use 'full' mode. Otherwise 'inline'.
6. Build response options:
   - 2-3 dialogue response options (generated from context)
   - If mode is 'full': add 1 check option `[心智检定 DC 12] 观察{npcName}的表情` with `requiresCheck: true, checkAttribute: 'mind', checkDc: 12`
   - Always add "结束对话" option
7. Update `dialogueStore.setState()` with all data
8. If `shouldRemember`: write memory to `npcMemoryStore`
9. Return `{ mode, dialogue: npcDialogue.dialogue, npcName }`

`processPlayerResponse(responseIndex: number): Promise<DialogueResult>`:
1. Get response from `dialogueStore.getState().availableResponses[responseIndex]`
2. If response is "结束对话": call `endDialogue()`, return
3. If `requiresCheck`: run adjudication with checkAttribute and checkDc
   - On success (grade >= 'success'): set `emotionHint` in dialogueStore based on AI's emotionTag per D-14
   - On partial_success: set generic hint `（{npcName}似乎{emotionTag}）`
   - On failure: no hint
4. Generate next NPC dialogue via `generateNpcDialogue` with playerAction = response label
5. Update dialogueStore with new dialogue line, new responses
6. If `shouldRemember`: write memory

`endDialogue()`:
1. Set `dialogueStore.setState(draft => { draft.active = false; draft.npcId = null; ... })`
2. Set `gameStore.setState(draft => { draft.phase = 'game'; })`

Layout per UI-SPEC Dialogue Mode:
```
│  【{npcName}】                  关系: {relationship label} ({value})    │
│                                                                         │
│  "{last dialogue line}"                                                 │
│                                                                         │
│  {emotionHint in dimColor italic if present, e.g., （他似乎在隐瞒什么）}│
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│  > 1. "{response 1}"                                                    │
│    2. "{response 2}"                                                    │
│    3. [心智检定 DC 12] 观察他的表情                                      │
│    4. 结束对话                                                          │
│                                                                         │
│  ↑↓ 选择    Enter 确认    Esc 结束对话                                   │
```

- NPC name: bold cyan, enclosed in `【】`
- Relationship: right-aligned on same row as NPC name, dimColor. Map value to label: < -0.5: '敌对', -0.5 to -0.1: '冷淡', -0.1 to 0.1: '中立', 0.1 to 0.5: '友好', > 0.5: '信任'
- Dialogue lines: body text, NPC speech in `"quotes"`
- Emotion hint: `<Text dimColor italic>（{hint}）</Text>` when present
- Response options: numbered list with `>` cursor, same interaction pattern as actions-panel.tsx
- Check options: prefix with `<Text color="yellow">[{checkAttribute}检定 DC {dc}]</Text>`
- Use `useInput` for keyboard: up/down navigation, Enter select, Esc ends dialogue
- Escape triggers `endDialogue` regardless of selected option

| AI NPC dialogue -> display | AI-generated dialogue text rendered to player |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-14 | Tampering | Player dialogue choices | mitigate | Player selects from pre-defined response options (indexed). Raw NL input in dialogue goes through intent classifier first. |
| T-02-15 | Information Disclosure | NPC leaking hidden info | mitigate | NPC system prompt constrains NPC to only discuss things they should know. Personality tags and knowledge boundaries are in system prompt. |
| T-02-16 | Repudiation | NPC memory writes | accept | Memory writes are append-only, logged via event bus. Single-player game, no dispute resolution needed. |
