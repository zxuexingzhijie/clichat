# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Core Gameplay. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This is a **terminal CLI** application using React + Ink. All values are in monospace character units, not pixels. Colors are ANSI terminal colors, not hex.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (terminal CLI, not web) |
| Preset | not applicable |
| Component library | Ink 7 (Box, Text) + @inkjs/ui (TextInput, Select) |
| Icon library | Unicode symbols only (no icon font) — `❯` cursor, `♥` HP, `✦` MP, `⚔` combat, `💀` death, `►` active |
| Font | Terminal monospace (user's terminal font — no control) |
| ASCII art | figlet with 'ANSI Shadow' font (established Phase 1) |
| Gradient | gradient-string cyan→magenta (established Phase 1) |
| CJK support | string-width for display width calculation (CJK char = 2 columns) |

---

## Spacing Scale

All values in **character units** (monospace columns/rows). Terminal UI has no sub-character precision.

| Token | Value | Usage |
|-------|-------|-------|
| none | 0 | Tight inline elements (label: value) |
| xs | 1 char | Standard paddingX for panels (`paddingX={1}`), icon-to-text gap |
| sm | 2 chars | Double-space separation between status bar fields |
| md | 3 chars | Section separation within a panel |
| row-gap | 1 row | Between text lines, between action items |
| section-gap | Divider row | `├───┤` divider between major sections (established Phase 1) |

Exceptions:
- Character creation wizard steps use 1 blank row between question and options
- Combat status bar uses `│` vertical separator between player stats and enemy stats
- Dialogue Mode uses 1 blank row between NPC speech and response options

Source: Established in Phase 1 codebase (`paddingX={1}` throughout, Divider component)

---

## Typography

Terminal monospace — no font-size control. Differentiation through Ink `<Text>` attributes only.

| Role | Technique | Ink Props | Usage |
|------|-----------|-----------|-------|
| Body | Normal text | `<Text>` (no props) | Scene narration, dialogue, descriptions |
| Label | Dim text | `<Text dimColor>` | Hints, inactive text, keyboard shortcuts, secondary info |
| Emphasis | Bold text | `<Text bold>` | Panel headings ("可选行动"), active selections, NPC names in dialogue, check results |
| Accent | Colored bold | `<Text bold color="cyan">` | Game title, primary CTA, input prompt, selected action, section headers |
| Danger | Red text | `<Text color="red">` | Critical HP (<10%), death, destructive confirmations, failed checks |
| Warning | Yellow text | `<Text color="yellow">` | Low HP (<25%), partial success, caution states, size guard warning |
| Success | Green text | `<Text color="green">` | Successful checks, HP healed, positive outcomes |
| Narrative | Italic (where supported) | `<Text italic>` | AI-generated narration text (degrades to normal in terminals without italic) |

Source: Phase 1 established `bold`, `dimColor`, `color="cyan"` patterns. Phase 2 adds green/red semantic colors for check results and combat.

---

## Color

ANSI terminal color palette. No hex values — terminal colors are named constants.

| Role | ANSI Color | Usage | Coverage |
|------|-----------|-------|----------|
| Dominant (60%) | Default terminal bg + default fg | Body text, scene narration, all standard content | Background surface, body text |
| Secondary (30%) | `dimColor` (terminal dim attribute) | Inactive actions, hints, keyboard shortcut labels, timestamps, secondary info | Hint text, inactive UI, separators |
| Accent (10%) | `cyan` | See reserved list below | Interactive highlights only |
| Destructive | `red` | Critical HP, death, flee failure, combat damage taken | Danger states only |
| Warning | `yellow` | Low HP, partial success, size guard, ambiguous outcomes | Caution states only |
| Positive | `green` | Successful checks, healed HP, quest complete, good outcomes | Success feedback only |
| Gradient | `cyan→magenta` | Title screen ASCII art only | Title screen only |

**Accent (`cyan`) reserved for:**
1. Game title in TitleBar (`Chronicle CLI`)
2. Input prompt character (`>` or `/`)
3. Currently selected action item in ActionsPanel
4. Active NPC name during Dialogue Mode
5. Character creation wizard — currently active step indicator
6. Combat — player's turn indicator

**Never use cyan for:** body text, narration, dividers, status bar values, dim hints.

Source: Phase 1 established cyan as sole accent. Phase 2 adds green (success) and widens red/yellow usage for combat/checks.

## Component Inventory — New for Phase 2

### 1. Character Creation Wizard

**Screen:** `CharacterCreationScreen` — replaces GameScreen during creation flow.

**Layout:** Full-screen within outer border. Vertical flow.

```
┌─ Chronicle CLI ─────────────────────── 角色创建 ─┐
│                                                   │
│  你从哪里来？                                      │
│                                                   │
│  ❯ 1. 人类 — 适应力极强，遍布大陆各地              │
│    2. 精灵 — 长寿而优雅，与自然和魔法有天然联系     │
│    3. 矮人 — （未来扩展）                          │
│                                                   │
│  ↑↓ 选择    Enter 确认                             │
│                                                   │
├─────────────────────────────────────────────────────┤
│  步骤 1/4  ■□□□                                    │
└─────────────────────────────────────────────────────┘
```

**Interaction contract:**
- 4 sequential steps: Race → Profession → Background Hook → Confirm
- Each step: narrative question (bold) + numbered option list with `❯` cursor (reuse ActionsPanel pattern)
- Step indicator at bottom: `步骤 N/4` + progress bar using `■□□□` characters
- Background hook step: 3-4 narrative questions that imply attribute distribution (D-03)
- Confirm step: summary of all choices + starting stats + equipment + "开始冒险" CTA
- Back navigation: Escape goes to previous step (except step 1, which goes to title screen)
- Quick mode: if not first playthrough, offer numbered presets before wizard starts

**State transitions:**
- Title screen → `onStart()` → Character creation (new) → Game screen
- Character creation → confirm → GameScreen with initialized player state

### 2. Dialogue Mode (within Scene Panel)

**Layout:** Scene panel content switches when entering Dialogue Mode (D-11, D-12, D-13).

**Inline dialogue** (short, non-critical):
```
│ 酒馆老板擦着杯子，头也不抬地说：                    │
│ "又一个找麻烦的冒险者。想喝什么？"                   │
```
Just rendered as narration lines in ScenePanel. No layout change.

**Dialogue Mode** (critical dialogue — quest, relationship, checks):
```
┌─ Chronicle CLI ──────────────────────── Day 1 / Night ─┐
│                                                         │
│  【守卫队长·陈铁柱】                  关系: 中立 (0)    │
│                                                         │
│  "黑松镇最近不太平。北林的狼群越来越多，              │
│   已经有两个樵夫失踪了。"                              │
│                                                         │
│  他皱着眉，语气中带着疲惫。                            │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  ❯ 1. "需要人帮忙吗？"                                 │
│    2. "那跟我有什么关系？"                              │
│    3. [心智检定 DC 12] 观察他的表情，判断是否有隐情     │
│    4. 结束对话                                          │
│                                                         │
│  ↑↓ 选择    Enter 确认    Esc 结束对话                  │
├─────────────────────────────────────────────────────────┤
│ HP 30/30  MP 8/8  Gold 12  Location: 黑松镇·北门        │
├─────────────────────────────────────────────────────────┤
│ >                                                       │
└─────────────────────────────────────────────────────────┘
```

**Component:** `DialoguePanel` — renders within ScenePanel area when dialogue mode active.

**Interaction contract:**
- NPC name: bold cyan, enclosed in `【】`
- Relationship status: right-aligned on same row as NPC name, dimColor
- NPC speech: body text, indented 1 char with opening `"` and closing `"`
- Narration/action descriptions: dimColor italic (NPC gestures, atmosphere)
- Emotion hint (after successful mind check per D-14): dim text in parentheses after narration line, e.g., `（他似乎在隐瞒什么）`
- Response options: numbered list with `❯` cursor (reuse ActionsPanel pattern)
- Check options: prefix with `[属性检定 DC N]` in yellow to indicate skill check required
- Esc exits dialogue mode, returns to normal scene view
- Response selection triggers: AI generates next NPC line + Rules Engine adjudicates any checks

### 3. Combat Mode (panel content swap)

**Layout:** All four panels remain, content changes per D-16.

```
┌─ Chronicle CLI ──────────────────── Day 1 / Night ─┐
│  ⚔ 战斗！                                          │
│                                                     │
│  [D20: 14] + 体魄 3 = 17 vs DC 15 → 成功！          │
│  你的剑划出一道弧光，斩中了灰狼的前腿。              │
│  灰狼发出一声哀嚎，踉跄后退。                       │
│                                                     │
│  灰狼龇牙咧嘴地扑了过来——                           │
│  [D20: 8] + 体魄 2 = 10 vs AC 14 → 失败             │
│  你侧身闪过，狼爪擦过你的肩膀。                     │
│                                                     │
├─────────────────────────────────────────────────────┤
│ ♥ 28/30  ✦ 8/8 │ 灰狼 ♥ 12/20 │ 回合 3 — 你的回合  │
├─────────────────────────────────────────────────────┤
│  ❯ 1. ⚔ 攻击 — 挥剑斩击                             │
│    2. ✦ 施法 — 火球术 (消耗 4 MP)                    │
│    3. 🛡 防御 — 提高闪避，下回合 AC+2                │
│    4. 🎒 物品 — 使用背包中的物品                     │
│    5. 🏃 逃跑 — [技巧检定 DC 10]                     │
│                                                     │
│  ↑↓ 选择    Enter 确认                               │
├─────────────────────────────────────────────────────┤
│ >                                                   │
└─────────────────────────────────────────────────────┘
```

**Panel changes in Combat Mode:**
- **Scene panel → Battle narration:** Check results (monospace formatted) + AI narration interleaved (D-17: check first, narration second)
- **Status bar → Combat status:** Player HP/MP + `│` separator + Enemy name + HP + `│` separator + Turn counter + whose turn
- **Actions panel → Combat actions:** Attack/Cast/Guard/Item/Flee with Unicode prefix icons, check DCs shown for flee/special
- **Input area → unchanged:** still accepts NL input for creative combat actions

**Interaction contract:**
- Combat entry: `⚔ 战斗！` header in bold at top of scene panel
- Check result format: `[D20: {roll}] + {attr_label} {mod} = {total} vs DC {dc} → {grade_label}` — matches Phase 1 D-20, using `GRADE_LABELS` from common.ts
- Success grades color: critical_success=green+bold, great_success=green, success=green, partial_success=yellow, failure=red, critical_failure=red+bold
- AI narration: renders after check result, body text, 80-180 Chinese chars (D-10)
- Enemy HP: show current/max, color follows same HP ratio rules as player (red <10%, yellow <25%)
- Turn indicator: `你的回合` (cyan, bold) vs `{enemy}的回合` (red)
- Combat exit: victory message (green) or defeat message (red) or flee success, then return to normal scene mode

### 4. Check Result Display

**Inline component:** `CheckResultLine` — renders within scene panel or combat narration.

Format: `[D20: {roll}] + {attrLabel} {mod} = {total} vs DC {dc} → {gradeLabel}`

| Element | Style |
|---------|-------|
| `[D20: {roll}]` | bold |
| `+ {attrLabel} {mod}` | normal |
| `= {total}` | bold |
| `vs DC {dc}` | dimColor |
| `→ {gradeLabel}` | colored by grade (see success grade colors above) |

Nat 20: entire line green + bold, append `（天命所归！）`
Nat 1: entire line red + bold, append `（命运弄人...）`

Source: Phase 1 D-20, Phase 2 D-17

### 5. Narration Display

AI-generated narration renders as body text in ScenePanel. No special container — just `<Text>` lines.

**Constraints:**
- 80-180 Chinese characters per turn (D-10)
- Never rendered with any "AI generated" indicator — it's narration, not a chatbot response
- Loading state while AI generates: `<Text dimColor>......</Text>` (6 dots, animating with Spinner from @inkjs/ui if latency > 500ms)
- If AI fails: `<Text color="red">叙述生成失败，请重试。</Text>` (error fallback, not empty)

## Interaction Patterns

### Input Mode Transitions

| From | Trigger | To |
|------|---------|-----|
| Title screen | Any key | Character creation (new game) or Game screen (existing save) |
| Character creation | Step confirm | Next step (or Game screen on final confirm) |
| Character creation | Esc | Previous step (or Title screen on step 1) |
| Action select mode | `/` key | NL/command input mode |
| NL/command input | Esc | Action select mode |
| Normal scene | `:talk NPC` (critical) | Dialogue Mode |
| Dialogue Mode | Esc or "结束对话" option | Normal scene |
| Normal scene | Combat trigger | Combat Mode |
| Combat Mode | Victory/defeat/flee | Normal scene |

### Keyboard Bindings

| Key | Context | Action |
|-----|---------|--------|
| `Up/Down` | Action select, Dialogue options, Combat actions, Char creation options | Navigate list |
| `Enter` | Any selection list | Confirm selection |
| `1-9` | Any selection list | Quick-select numbered option |
| `/` | Action select mode | Switch to text input mode |
| `Esc` | Text input, Dialogue Mode, Char creation | Back/cancel |
| `Y/N` | Confirmation prompts | Confirm/deny destructive action |
| Any key | Title screen | Start game |

### Responsive Breakpoints

| Width | Behavior | Source |
|-------|----------|--------|
| < 80 cols | Size guard — "终端窗口过小" | Established Phase 1 |
| 80-99 cols | Narrow layout — vertical stack (all panels stacked) | Established Phase 1 |
| >= 100 cols | Wide layout — scene panel 60% left, actions 40% right | Established Phase 1 |

Combat Mode and Dialogue Mode follow the same breakpoint rules. In narrow mode, combat status bar may truncate enemy name.
