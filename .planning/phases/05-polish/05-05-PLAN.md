  <behavior>
    Worker (summarizer-worker.ts):
    - runSummarizerLoop(): called once at startup; runs an async loop that:
      1. Checks dequeuePending() — if null, sleeps 5s (setTimeout/Promise) and loops
      2. If task found: markRunning(task.id), calls dispatchTask(task), then markDone or markFailed
      3. Non-blocking: never awaited by game loop; called via `void runSummarizerLoop()` at init
    - dispatchTask(task): routes to the correct LLM function by task.type
    - applyNpcMemoryCompression(task, result): version check before write-back:
      ```
      const record = npcMemoryStore.getState().memories[task.targetId]
      if (!record || record.version !== task.baseVersion) return 'conflict'
      npcMemoryStore.setState(draft => {
        const r = draft.memories[task.targetId]
        if (r) {
          r.archiveSummary = result
          r.recentMemories = r.recentMemories.slice(task.entryIds.length)
          r.version += 1
        }
      })
      return 'applied'
      ```
    - On conflict: markFailed(task.id) and do NOT re-queue (preserve originals)
    - On LLM error: markFailed(task.id), do NOT re-queue (user can trigger manually)

    Prompts (summarizer-prompts.ts):
    - buildNpcMemoryCompressPrompt(npcId, memories): returns system + user prompt pair for compressing NpcMemoryEntry[] into a single archiveSummary string. Chinese output. Max 200 chars.
    - buildChapterSummaryPrompt(recentNarration): returns prompt for a chapter summary narrative (max 300 chars). Chinese.
    - buildTurnLogCompressPrompt(turns): returns prompt for summarizing TurnLogEntry[] into a 100-200 char block. Chinese.

    - All three use generateText (not streamText — background task, no streaming needed)
    - Error handling: on failure, throw — worker catches and marks task as failed

| Boundary | Description |
|----------|-------------|
| NpcMemoryEntry.event string → LLM prompt | Game-generated content; passes through safety-filter upstream; low risk |
| LLM response → NPC archive | AI-generated text stored in game state; rendered as plain text only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Tampering | applyNpcMemoryCompression version check | mitigate | Version must equal baseVersion exactly; mismatch → preserve originals, no overwrite |
| T-05-11 | Denial of Service | runSummarizerLoop infinite loop | mitigate | 5s sleep between iterations; max 1 concurrent task; combat gate prevents firing during gameplay |
| T-05-12 | Tampering | summarizer prompt injection | accept | NPC memory events are game-generated (not user-controlled freeform text); no direct player input in summarizer prompts |
</threat_model>
