# Phase 8 — UI Design Contract

> Visual and interaction contract for the narrative character creation screen. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (CLI terminal — shadcn not applicable) |
| Preset | not applicable |
| Component library | Ink 7 (`<Box>`, `<Text>`) + @inkjs/ui (`TextInput`) |
| Icon library | Unicode characters (`❯`, `─`, `├`, `┤`) — existing pattern |
| Font | Terminal monospace (system default) |

**Rationale:** This is a terminal CLI game rendered via React + Ink. All "styling" is ANSI escape codes via Ink's `color`, `bold`, `dimColor` props. No CSS, no Tailwind, no shadcn.

---

## Spacing Scale

Terminal spacing uses character cells, not pixels. All values below are in **terminal cells** (1 cell = 1 monospace character width, 1 row height).

| Token | Value | Usage | Source |
|-------|-------|-------|--------|
| inline | `paddingX={1}` (1 cell) | All panel content inset from border | Existing: ScenePanel, ActionsPanel, StatusBar, DialoguePanel |
| gap-sm | `marginTop={1}` (1 row) | Between related content sections | Existing: TitleScreen, CharacterCreationScreen |
| gap-md | `marginTop={2}` (2 rows) | Not used — single blank `<Text> </Text>` lines used instead | Existing: DialoguePanel uses `<Text> </Text>` |
| separator | `<Divider width={innerWidth} />` | Between layout regions (scene/status/actions/input) | Existing: GameScreen layout |
| blank-line | `<Text> </Text>` or `<Text>{''}</Text>` | Visual breathing room within a panel | Existing: DialoguePanel |

Exceptions: none — follow existing patterns exactly.

---

## Typography

Terminal monospace has no font size control. "Typography" in this context means Ink text attribute combinations.

| Role | Ink Props | Usage | Source |
|------|-----------|-------|--------|
| Body | `<Text>` (no props) | Narration lines, guard dialogue text, player dialogue history | ScenePanel lines |
| Heading | `<Text bold>` | Section headers ("可选行动"), guard question text | ActionsPanel header, CharacterCreationScreen question |
| Label-accent | `<Text bold color="cyan">` | NPC name header, game title, combat header | DialoguePanel `【{npcName}】`, TitleBar, GameScreen |
| Label-secondary | `<Text dimColor>` | Hints, help text, relationship display, progress indicators | Ubiquitous across all panels |
| Emphasis | `<Text color="yellow">` | Selected/highlighted option text, stat values in character preview | CharacterCreationScreen options, stat display |
| Streaming-cursor | `<Text dimColor>...</Text>` | Appended to streaming text while AI is generating | ScenePanel streaming indicator |

Weight constraint: only 2 weights used — `bold` (true) and normal (default). No other weights exist in terminal.

---

## Color

ANSI named colors only. No hex values — terminal color interpretation varies by user theme.

| Role | ANSI Color | Ink Prop | Usage | Source |
|------|------------|----------|-------|--------|
| Dominant (60%) | default terminal foreground | `<Text>` | Body text, narration, dialogue content | All panels |
| Secondary (30%) | dim | `dimColor` | Hints, inactive options, progress bars, help text | All panels |
| Accent (10%) | cyan | `color="cyan"` | Reserved for: NPC name headers, game title bar, selected option highlight, active prompt marker | TitleBar, DialoguePanel, ActionsPanel |
| Selection highlight | yellow | `color="yellow"` | Reserved for: currently-highlighted option (before Enter), stat value callouts, skill check badges | CharacterCreationScreen cursor, DialoguePanel check DC |
| Confirm | green | `color="green"` | Reserved for: final confirmation CTA only | CharacterCreationScreen "开始冒险" |
| Danger/HP | red | `color="red"` | HP display when low, error messages | StatusBar hpColor |
| MP | blue | `color="blue"` | MP stat display | CharacterCreationScreen preview |

**Phase 8 accent usage (cyan):** Guard NPC name header `【北门守卫】`, active prompt indicator `> ` in name input mode.

**Phase 8 selection highlight (yellow):** Dialogue option the cursor is currently on.

**Green is NOT used in Phase 8.** There is no confirmation screen (D-04: seamless transition). Do not render a green "confirm" button.

## Interaction Contract

### Dialogue Round Interaction (rounds 1-4)

| Input | Behavior | Active When |
|-------|----------|-------------|
| Up arrow | Move selection cursor up (wrap to bottom) | Options visible, not streaming |
| Down arrow | Move selection cursor down (wrap to top) | Options visible, not streaming |
| Enter | Confirm selected option, trigger next guard response (AI streaming) | Options visible, not streaming |
| Number keys 1-N | Direct-select option N and confirm immediately | Options visible, not streaming |
| Enter or Space | Skip streaming animation to show full text | Guard response is streaming |

### Name Input Interaction (after round 4)

| Input | Behavior | Active When |
|-------|----------|-------------|
| Typing | Free-text name entry via @inkjs/ui TextInput | Name input mode active |
| Enter | Confirm name (or '旅人' if empty) | Name input mode active |
| Tab | Generate and fill a random name suggestion | Name input mode active |

### Farewell Transition Interaction

| Input | Behavior | Active When |
|-------|----------|-------------|
| Enter or Space | Skip farewell streaming to show full text, then transition | Farewell is streaming |
| (auto) | After farewell text completes (streamed or skipped), auto-transition to `game` phase after 500ms delay | Farewell complete |

### Navigation Constraints

| Constraint | Rule |
|------------|------|
| No back navigation | Cannot go back to previous dialogue rounds (D-04: seamless flow, no confirmation) |
| No escape to title | No Escape handler during narrative creation — player is committed once started |
| No overlay panels | No map, journal, codex, shortcuts during creation — those activate only in `game` phase |
| No free-text input during dialogue rounds | Only fixed options during rounds 1-4. Free-text only for name. |

## State Machine

```
title (press any key)
  → narrative_creation
    → round_1_streaming (guard asks origin question via AI)
      → round_1_selecting (player picks from fixed options)
        → round_2_streaming (guard reacts + asks livelihood)
          → round_2_selecting
            → round_3_streaming (guard reacts + asks reason for visiting)
              → round_3_selecting
                → round_4_streaming (guard reacts + asks secret)
                  → round_4_selecting
                    → name_prompt_streaming (guard asks name)
                      → name_input (free-text input active)
                        → farewell_streaming (guard waves through, AI farewell)
                          → transition_delay (500ms)
                            → game (normal game loop)
```

Each `*_streaming` state: guard narration area shows streaming text with `...` cursor. Options area hidden or shows "Enter/Space 跳过" hint.

Each `*_selecting` state: guard narration area shows completed text. Options area shows fixed choices with arrow/Enter interaction.
