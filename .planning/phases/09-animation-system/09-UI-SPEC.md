# Phase 9 — UI Design Contract

> Visual and interaction contract for the Animation System phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (terminal UI -- no CSS framework applicable) |
| Preset | not applicable |
| Component library | React 19 + Ink 7 + @inkjs/ui 2.0.0 |
| Icon library | Unicode symbols (inline text characters: `♥`, `✦`, `⚔`, `❯`, `│`) |
| Font | Terminal default (monospace, determined by user's terminal emulator) |

**Note:** This is a CLI application rendered via Ink's alternate screen buffer. All "styling" is Ink `<Text>` props (`color`, `bold`, `dimColor`, `inverse`) and `<Box>` layout props (`paddingX`, `marginTop`, `flexDirection`, `flexGrow`). No pixel-based values apply.

---

## Spacing Scale

Terminal UI uses character cells, not pixels. Ink spacing is expressed as `paddingX={N}` (character columns) and `marginTop={N}` (rows).

| Token | Value | Usage |
|-------|-------|-------|
| none | 0 | Inline elements, tight stacking |
| xs | paddingX={1} | Standard panel horizontal padding (established pattern) |
| sm | marginTop={1} | Section separation within panels (established pattern) |
| md | marginTop={2} | Major section breaks (used in CodexPanel detail) |
| gap | `<Text>{'  '}</Text>` | Inline field separation in status bars (2-space gap) |

Exceptions: Toast banner uses `paddingX={1}` with no extra margin (sits at top of ScenePanel content area). Title screen uses `marginTop={1}` between title art, subtitle, and prompt.

Source: Established patterns in status-bar.tsx, codex-panel.tsx, scene-panel.tsx.

---

## Typography

Terminal UI has no font-size control. All text is the terminal's monospace font at the user's configured size. "Typography" maps to Ink `<Text>` formatting props.

| Role | Ink Props | Usage |
|------|-----------|-------|
| Body | `<Text>` (no props) | Scene narration lines, dialogue text, panel content |
| Label | `<Text dimColor>` | Hint text, inactive fields, metadata, "Esc 返回" hints |
| Emphasis | `<Text bold>` | Panel titles, selected items, key values (HP, roll results) |
| Accent | `<Text color="cyan" bold>` | Panel headers ("【知识典籍】"), active turn indicator, title fallback |
| Flash (damage) | `<Text color="red" bold>` | HP damage flash, critical failure, 300ms duration |
| Flash (heal) | `<Text color="green" bold>` | HP heal flash, critical success, 300ms duration |
| Gradient | `gradient-string('cyan', 'magenta')` | Title screen ASCII art (pre-computed per-character colors) |
| Dim | `<Text dimColor>` | Fade-out state (scene transition, spinner dimout, subtitle pre-reveal) |

Source: Established patterns in title-screen.tsx, status-bar.tsx, check-result-line.tsx, codex-panel.tsx, combat-status-bar.tsx.

## Color

Terminal ANSI color palette. The 60/30/10 model maps to terminal text styling frequency.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Default terminal foreground | Body text, narration, dialogue, panel content |
| Secondary (30%) | `dimColor` (ANSI dim attribute) | Labels, hints, inactive fields, metadata, separators, fade-out states |
| Accent (10%) | `cyan` | Panel headers, active turn indicator ("你的回合"), selected item highlight, title fallback text |
| Destructive | `red` | HP critical (<10%), damage flash, critical failure, enemy turn indicator, quest failed toast |
| Success | `green` | HP heal flash, success/critical success grade, quest completed toast, knowledge discovered toast |
| Warning | `yellow` | HP low (<25%), partial success grade, gold change toast |
| Info | `blue` | Knowledge status badges ("传闻", "推断"), related entry links |
| Gradient start | `cyan` | Title ASCII art gradient left edge |
| Gradient end | `magenta` | Title ASCII art gradient right edge |

Accent reserved for: Panel headers (e.g. "【章节总结】"), active selection highlight (`color="cyan"`), active turn indicator in combat, title screen fallback. Never used for body text or generic decorations.

Source: Established color semantics from CONTEXT.md code_context section, verified against status-bar.tsx, check-result-line.tsx, codex-panel.tsx, combat-status-bar.tsx.

## Animation Timing Contract

All timings are locked from CONTEXT.md decisions D-01 through D-18.

### Title Typewriter (ANIM-01)

| Property | Value | Source |
|----------|-------|--------|
| Character reveal interval | 20-30ms per character | D-02 |
| Total reveal duration | ~1-1.5s for "CHRONICLE" | D-02 |
| Gradient application | Progressive -- pre-computed per-character, color stable from moment of reveal | D-01 |
| Reveal direction | Left-to-right, column-by-column across multi-line figlet block | D-01 |
| Skip trigger | Any key press | D-03 |
| Skip behavior | Immediately render full title with complete gradient | D-03 |
| Post-reveal | Subtitle + menu fade-in from dim to bright over 2-3 steps | D-04 |
| Subtitle fade | `dimColor={true}` -> `dimColor={false}` over ~250ms (2-step) | D-04 |
| Plays on | Every launch (no first-launch-only flag) | D-02 |

### AI Loading Spinner (ANIM-02)

| Property | Value | Source |
|----------|-------|--------|
| Component | `@inkjs/ui` `<Spinner>` | D-05 |
| Spinner type | `dots` (Claude's discretion -- dots is clean, minimal, fast cycle) | Discretion |
| Position | Inside ScenePanel where streaming text will appear | D-05 |
| Labels by context | narration: "命运之轮转动中..." / "史官正在记录..."; npc: "正在思考..."; combat: "攻击展开..." / "局势变化..." | D-06 |
| Label selection | Random from context pool on each activation | D-06 |
| Activation | On AI call initiation (before streaming_started event) | D-08 |
| Deactivation | When first streaming chunk arrives | D-08 |
| Transition | Spinner dims (`dimColor={true}`) for ~150ms, brief pause, then first streaming sentence appears | D-07 |
| State machine | `IDLE -> SPINNER_ACTIVE -> SPINNER_DIMMING -> STREAMING -> IDLE` | Research recommendation |

### Scene Transition (ANIM-03)

| Property | Value | Source |
|----------|-------|--------|
| Trigger | `scene_changed` event | D-09 |
| Fade-out | Old content `dimColor={true}` for ~250ms | D-09 |
| Fade-in | New content starts `dimColor={true}`, transitions to `dimColor={false}` after ~250ms | D-09 |
| Total duration | ~500ms | D-09 |
| Implementation | Single state variable: `FadeState = 'dim' | 'normal' | 'hidden'` | Research Pattern 5 |

### Combat HP Flash (ANIM-04)

| Property | Value | Source |
|----------|-------|--------|
| Damage trigger | `player_damaged` event (+ `damage_dealt` with enemy targetId for combat) | D-12 |
| Heal trigger | `player_healed` event | D-12 |
| Damage visual | Name + HP value flash `color="red" bold` for 300ms | D-10 |
| Heal visual | Name + HP value flash `color="green" bold` for 300ms | D-10 |
| Flash scope | Name + HP value of affected entity ONLY, not entire status bar row | D-11 |
| Applies to | `status-bar.tsx` (exploration) AND `combat-status-bar.tsx` (combat mode) | D-10 |
| Revert | Returns to normal color/bold state after 300ms | D-10 |

### Check Result Flash (ANIM-05 partial)

| Property | Value | Source |
|----------|-------|--------|
| Trigger | On render of CheckResultLine component | D-13 |
| Visual | Brief `bold` flash enhancement for ~300ms after result renders | D-13 |
| Nat 20 / Nat 1 | Already have color coding -- flash adds temporal emphasis on top | D-13 |
| Implementation | `useTimedEffect(300)` triggered on mount | Research Pattern 1 |

### Toast Notifications (ANIM-05)

| Property | Value | Source |
|----------|-------|--------|
| Position | Single-line colored banner at top of ScenePanel content area | D-14 |
| Dismiss | Auto-dismiss after 2 seconds (default), max 3 seconds for important events | D-14 |
| Stacking | None -- single-replacement mode: new toast replaces existing one | D-14 |
| Layout | `<Box paddingX={1}><Text color={color} bold>{icon} {message}</Text></Box>` | Discretion |
| Border | None (single-line, no box border -- keeps it lightweight) | Discretion |

### Toast Event Map

| Event | Icon | Color | Message Template | Dismiss (ms) |
|-------|------|-------|------------------|---------------|
| `quest_started` | `!` | `cyan` | "新任务: {questTitle}" | 2000 |
| `quest_completed` | `*` | `green` | "任务完成: {questTitle}" | 3000 |
| `quest_failed` | `x` | `red` | "任务失败: {questTitle}" | 3000 |
| `knowledge_discovered` | `?` | `blue` | "发现新知识: {entryId}" | 2000 |

| `summarizer_task_completed` (chapter) | `#` | `cyan` | "新章节总结可查看" | 3000 |

## Interaction States

### Title Screen Flow

```
[Launch] -> TYPEWRITER_REVEAL (chars appearing 20-30ms each)
   |-- any key -> SKIP (full title + gradient immediately)
   |-- complete -> SUBTITLE_FADE_IN (dim -> bright, ~250ms)
   |-- SUBTITLE_FADE_IN complete -> MENU_VISIBLE (interactive)
   |-- any key during MENU_VISIBLE -> onStart()
```

### Spinner-to-Streaming Flow

```
[AI call initiated] -> SPINNER_ACTIVE (Spinner visible with label)
   |-- first streaming chunk arrives -> SPINNER_DIMMING (dimColor=true, ~150ms)
   |-- SPINNER_DIMMING timeout -> STREAMING (spinner removed, text rendering)
   |-- streaming complete -> IDLE
```

### Scene Transition Flow

```
[scene_changed event] -> FADE_OUT (current content dimColor=true, ~250ms)
   |-- timeout -> CONTENT_SWAP (new scene content loaded)
   |-- -> FADE_IN (new content dimColor=true -> dimColor=false, ~250ms)
   |-- timeout -> NORMAL
```

### Toast Flow

```
[domain event fires] -> TOAST_VISIBLE (banner at top of ScenePanel)
   |-- auto-dismiss timer (2-3s) -> TOAST_HIDDEN
   |-- new event before dismiss -> TOAST_REPLACED (old dismissed, new shown)
```
