From src/ai/roles/safety-filter.ts (current pattern — line 6):
```typescript
const STATE_OVERRIDE_PATTERN = /(获得|失去|HP|MP|金币|等级|升级|gained|lost|level\s*up|gold|experience)\s*[+\-]?\d+/i;
```

Problem: matches "金币100" or "金币+10" — but the false positive arises from narration like
"你获得了+10金币" where `获得` is followed (after 了) by `+10`. The `\s*[+\-]?\d+` matches:
`获得` → then `\s*` (0) → `[+\-]?` (0) → `\d+` fails on `了`. BUT if narration is
"你获得了 +10 金币" the space allows: wait — `了` is still not whitespace.
The real false-positive case is likely: "获得10" without 了, or "金币10" at end.
The fix: require [+\-] to be PRESENT (not optional) for the word-based triggers OR require
a non-Chinese-character boundary. The safest minimal fix: change `[+\-]?` to `[+\-]` (required)
for the attack — this means "获得10" no longer matches, but "获得+10" and "获得-10" still do.
This preserves all injected patterns (attackers write "+100 HP" style) while not catching
narrative prose ("你获得了10枚金币" contains "获得" but no operator before the number).
