<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| keyboard → InputProvider | Raw player key input controls game UI and command submission |
| EventBus → state machine | Domain events switch keyboard context |
| InputProvider → Rules Engine/AI | Provider dispatches player actions to deterministic systems and then narrative streaming |
| AI streaming state → input global layer | Streaming state changes whether Enter/Space skips or dispatches action |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-05-01 | T | keyboard dispatch | mitigate | Dual-layer handler consumes global keys first; exactly one state-level handler active via `isActive` guards. |
| T-22-05-02 | S | EventBus transitions | accept | Events are in-process domain events, not external auth claims; state machine only changes UI input context. |
| T-22-05-03 | E | AI stream influence | mitigate | `useIsStreaming()` only enables skip/disable behavior; AI text never selects actions or changes game state. |
| T-22-05-04 | D | input handler leakage/double-fire | mitigate | D-08 independent active handlers prevent combat/dialogue/menu double-fire; tests assert inactive states ignore keys. |
| T-22-05-05 | T | branch destructive action | mitigate | BRANCH handler must preserve confirmation `(y/N)` before switch; no single-key destructive switch. |
</threat_model>
