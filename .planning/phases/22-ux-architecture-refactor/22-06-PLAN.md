<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App computed overlay data → InputProvider | Map/codex/branch data crosses into provider selector state and must remain read-only UI data |
| InputProvider currentState → panel-local useInput handlers | State-machine context determines which mounted overlay may process keyboard input |
| Branch overlay input → branch switching | Branch tree can trigger destructive branch switch and must retain confirmation before switching |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-22-06-01 | T | `useOverlayPanelData` | mitigate | Expose App-provided overlay data directly; do not fabricate placeholder map/codex/branch data in the provider or GameScreen. |
| T-22-06-02 | D | overlay keyboard handlers | mitigate | Gate panel-local `useInput` handlers with provider `currentState` so inactive overlays cannot double-handle keys. |
| T-22-06-03 | T | `BranchTreePanel` switch | mitigate | Preserve existing confirm-before-switch flow; no single-key branch switch. |
| T-22-06-04 | I | codex/branch panel display | accept | Existing panels already render game-local world/save data; this plan only restores data flow lost during refactor. |
