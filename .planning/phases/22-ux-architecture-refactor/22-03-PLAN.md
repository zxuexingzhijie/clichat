<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI stream → NarrativeProvider | AI-generated narration/dialogue enters UI state as untrusted prose |
| NarrativeProvider → InputProvider | Derived streaming state controls input behavior but must not expose AI authority over game state |
| Provider hooks → renderer | Renderer receives text/errors and must display recovery copy, not raw internal stack traces |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-03-01 | I | stream errors | mitigate | Expose `Error` objects for controller handling; renderer-level copy must use Chinese recovery text, not stack traces. |
| T-22-03-02 | T | AI stream content | mitigate | Preserve existing separation: AI prose updates narration/dialogue text only; Rules Engine/controller still owns state changes. |
| T-22-03-03 | D | long/failed streams | mitigate | Preserve `skipToEnd()` and `reset()` hooks; InputProvider will subscribe to `useIsStreaming()` in Plan 05. |
| T-22-03-04 | E | provider context absence | mitigate | Selector hooks throw `ReferenceError` outside NarrativeProvider. |
</threat_model>
