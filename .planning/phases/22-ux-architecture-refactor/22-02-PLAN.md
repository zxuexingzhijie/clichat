<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| EventBus → AtmosphereProvider | Domain events update UI toast/dimout state and must not carry executable behavior |
| World/codex quest data → UI provider | Quest titles/tags are untrusted-ish content displayed in terminal UI |
| Provider hooks → GameScreen | Missing provider context should fail fast instead of silently using stale/global state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-02-01 | T | `AtmosphereProvider` event handlers | mitigate | Subscribe/unsubscribe exact handler references in `useEffect` cleanup; tests assert cleanup. |
| T-22-02-02 | I | toast messages | mitigate | Display concise player-facing strings only; never expose raw stack traces or secret values in toast. |
| T-22-02-03 | D | repeated dimout/toast timers | mitigate | Use Clock-backed `createTimedEffect`/toast reset behavior so repeated triggers clear previous timers. |
| T-22-02-04 | E | provider context absence | mitigate | Hooks throw `ReferenceError` when used outside AtmosphereProvider, matching existing store context pattern. |
</threat_model>
