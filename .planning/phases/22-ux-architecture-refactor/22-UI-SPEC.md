## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline symbol gaps only when represented as literal spaces inside text |
| sm | 8px | Default terminal padding: `paddingX={1}`, one blank row, one leading indentation unit |
| md | 16px | Panel grouping: two-cell indentation, two-row separation around major dialogue/action sections |
| lg | 24px | Reserved vertical rhythm for mode transitions and overlay headers; avoid in persistent gameplay chrome |
| xl | 32px | Major mode breaks only; not used in the always-on game screen stack |
| 2xl | 48px | Not used in Phase 22 implementation |
| 3xl | 64px | Not used in Phase 22 implementation |

Exceptions: Terminal rendering is cell-based, not pixel-based. In Ink code, `paddingX={1}` is the canonical `sm` spacing unit; do not introduce fractional cell spacing or CSS-like layout concepts. Existing compact 4-panel stack remains: Title bar (1 row) → NarrativeRenderer flex area → Status bar (1 row) → Actions/State controls (3–7 rows) → Input row (1 row).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px terminal default | 400 / regular | 1.5 equivalent; one terminal row per wrapped line |
| Label | 14px terminal default | 400 / regular | 1.5 equivalent; dim secondary labels and hints |
| Heading | 20px terminal default | 600 / semibold via `<Text bold>` | 1.2 equivalent; one row for panel headers |
| Display | 28px figlet/title only | 600 / semibold via figlet/bold | 1.2 equivalent; not used inside Phase 22 gameplay refactor |

**Typography contract:** use exactly two weights: regular and bold/semibold. Do not add italic as a required state cue; existing italic emotion hints may remain but must not carry unique meaning without text.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Terminal default background + default white ink | Scene/narrative body, stable gameplay chrome, normal NPC dialogue |
| Secondary (30%) | `dimColor` / gray ink | Hints, timestamps, relationship labels, scroll indicators, inactive actions, atmosphere text |
| Accent (10%) | `cyan` | Selected action marker, active prompt, current panel title, focused interactive key labels, active NPC name/glyph |
| Destructive | `red`; `bold red` only for 朱砂红 scarcity cases | Error text, failed checks, damage; bold red reserved for HP < 20%, NPC betrayal, player death |

Accent reserved for: `❯` selected row marker, active input prompt (`> ` or `/ `), active panel title, focused shortcut keys, active NPC speaker header/glyph, current map/location marker. Never use cyan for passive decoration, static body copy, or all interactive elements.

**Semantic color inheritance:** retain existing `green` for success/recovery/completion, `yellow` for warning/low resources/check DC labels, `red` for failure/damage. Phase 22 must not implement the Phase 24 Delight Layer as new behavior, but it must preserve compatibility with `DESIGN.md` 墨分五色 hierarchy and the future 朱砂红 restriction.

## Phase 22 Interaction Contract

### Scope Boundary

Phase 22 is an architecture refactor with user-visible behavior preservation plus clearer mode cues. It must not redesign the product, implement Phase 24 animations, add web UI patterns, or change the CLI-first 4-panel layout.

### NarrativeRenderer Contract

`NarrativeRenderer` replaces `ScenePanel` in-place and becomes the single narrative surface.

| Mode | Visual Contract | Interaction Contract |
|------|-----------------|----------------------|
| Exploration | Render scene lines as body text; system/weather tags use secondary/warning styling; streaming text appears inline with dim ellipsis | PgUp/PgDn scroll history when content exceeds viewport; auto-stick to bottom unless user scrolled up |
| Dialogue | Render embedded `DialogueView` within the same `NarrativeRenderer` instance; header format: `【glyph? NPC Name】` in cyan/bold, relationship label dim right-aligned | No parent remount when entering/exiting dialogue; ↑/↓ selects response, Enter confirms, Tab toggles recent/all dialogue history, direct typing enters free-text reply, Esc exits free-text first then dialogue |
| Combat narration | Render last combat narration/check result in narrative surface without replacing the whole layout stack | Combat action handler owns combat keys; narrative surface remains readable and scrollable where applicable |
| Processing/streaming | Show existing spinner/dimout pattern before text arrives; once text streams, spinner yields to streaming text | Enter/Space skips active narration or NPC dialogue stream; regular action execution is disabled while stream is active |
| Empty | Show `周围一片寂静。` body and next-step hint | Do not trap focus; action/input layer remains available |
| Error | Show short Chinese problem + recovery path using red regular text; only fatal boundary title may be red/bold | Esc should attempt to return to prior safe state when state machine can recover |

### 7-State Input State Machine Contract

Only one state-level `useInput` handler may be active at a time. Global layer handles cross-state keys first.

| State | Visible Cue | Active Keys | Disabled/Blocked Behavior |
|-------|-------------|-------------|----------------------------|
| EXPLORATION | Action panel title `可选行动`; selected row uses cyan `❯`; input prompt `> ` | ↑/↓, number keys, Enter, Tab or `/` for command input, direct NL input, panel shortcuts `m/j/c/i/b/?` | During streaming, Enter/Space skip stream instead of executing a new action |
| DIALOGUE | NarrativeRenderer shows NPC header + relationship; hint row: `↑↓ 选择    Enter 确认    Tab 全部/最近对话    直接输入 与NPC对话    Esc 结束对话` | ↑/↓, number keys, Enter, Tab, direct typing, Esc | Global panel shortcuts do not open overlays while typing or while DialogueView free-text is active |
| COMBAT | Action panel title `战斗行动`; waiting labels `敌方行动中...` / `处理中...` dim | ↑/↓, number keys, Enter for enabled combat actions; Esc handled by global help/cancel if defined | Disabled actions remain dim and do not execute; insufficient MP reason appears red only on selected disabled row |
| MENU | Overlay/panel title in cyan; `Esc 返回` dim on the right | Panel-specific navigation and Esc | Gameplay action keys do not leak into menu panels |
| CODEX | `【知识典籍】` or existing Codex panel title; selected entry cyan | Existing Codex navigation + Esc | Exploration shortcuts inactive except global `?`/Ctrl-C |
| MAP | Map panel title/current location cyan; non-current locations default/dim | Arrow navigation + Esc | Scene scroll PgUp/PgDn does not also move map selection |
| BRANCH | Branch tree/compare panel title; destructive switch confirmation uses yellow `(y/N)` | Tree navigation, compare/switch keys, y/n confirmation, Esc | Branch switch requires confirmation copy above; no immediate destructive action from a single key |

**Global layer:** Ctrl-C always exits through Ink app exit path; `?` opens shortcut help unless a text input owns typing; Esc closes overlay/free-text/current cancellable state before exiting gameplay. Global layer consumes first; state handlers run only if global did not consume.

### Provider UI Responsibilities

| Provider | Owns UI-Relevant State | Exposed Hooks | Visual Contract |
|----------|------------------------|---------------|-----------------|
| AtmosphereProvider | scene atmosphere tags, time of day, weather, quest ecological context, toast events, spinner/dimout effects | `useAtmosphere()`, `useToast()`, `useActiveQuests()` | Supplies title/status labels, toast banner data, spinner context, scene dimming; no component should recalculate active quest display independently |
| NarrativeProvider | narration stream, NPC dialogue stream, scene text, narration errors, skip/reset behavior | `useNarrationStream()`, `useDialogueStream()`, `useNarrativeText()`, derived `useIsStreaming()` | Single source of truth for displayed narrative and dialogue streaming; InputProvider subscribes for skip/disable behavior |
| InputProvider | current input state, input value, selected action indexes, state machine transition handling, controller action dispatch | `useInputState()`, `useInputActions()`, `useSelectedAction()`, `useCommandInput()` | Ensures only active mode handles keys; publishes mode cues to panels; controller logic no longer lives in GameScreen |

Provider nesting default: Atmosphere → Narrative → Input → GameScreen, unless implementation discovers a stricter dependency order. GameScreen remains a layout orchestrator under 100 lines and must not own domain calculations, streaming, or keyboard dispatch.

### Layout Contract

- Preserve the current terminal stack and responsive behavior.
- `NarrativeRenderer` owns the flexible vertical region and must accept `height` constraints from `fullscreen-ink`.
- Status bar remains one row and truncates CJK-safe with `string-width`.
- Actions/control region remains below status; do not move primary actions into the narrative body.
- Overlay panels (`journal`, `map`, `codex`, `inventory`, `branch_tree`, `compare`, `shortcuts`, `replay`, `chapter_summary`) may replace the narrative region, but must not remount global providers.

### Accessibility & Terminal Compatibility

- Every color-coded state must also have text/glyph indication (`❯`, labels, headings, warning copy).
- No interaction may require mouse input.
- Unicode glyphs must be width-safe; if width is uncertain, fallback to ASCII/`○`.
- Maintain high contrast on dark terminals; rely on Ink named colors and `dimColor`, not custom RGB gradients for Phase 22.
- CJK copy must not assume character count equals terminal width; use `string-width` for truncation and alignment.
