# Roadmap: Chronicle

## Overview

Chronicle delivers an AI-driven CLI text RPG where a deterministic Rules Engine controls truth and outcomes while AI generates narration and NPC personality. The roadmap moves from infrastructure (Rules Engine, CLI shell, state management) through core gameplay (explore, fight, talk) to persistence (save/load, memory, quests) to differentiation (branching, maps, epistemic separation) to polish (summarization, multi-provider routing, cost tracking). Each phase delivers a playable increment that proves the next layer of the product thesis.

## Phase Details

### Phase 1: Foundation
**Goal**: The game engine exists -- commands are parsed, a Rules Engine adjudicates outcomes deterministically, game state is a serializable immutable tree, the CLI renders a four-panel layout, and World Codex YAML schema is defined and validated
**Depends on**: Nothing (first phase)
**Requirements**: CORE-01, CORE-02, CORE-03, CORE-04, CLI-01, WORLD-01
**Success Criteria** (what must be TRUE):
  1. Player can type structured commands (`/look`, `/go north`) and see them parsed into game actions displayed in the CLI
  2. Player can type free-form natural language and see it classified into a structured intent
  3. Rules Engine resolves a test action (e.g., stat check) deterministically with zero LLM involvement, and the result appears in the scene panel
  4. Game state can be serialized to JSON and restored from JSON with identical state
  5. Terminal displays four-panel layout (scene, status bar, suggested actions, input) that responds to resize

### Phase 2: Core Gameplay
**Goal**: A player can create a character, explore scenes, talk to NPCs with AI-generated personality-driven dialogue, and fight enemies -- the core gameplay loop (create -> explore -> interact -> fight) is playable end-to-end
**Depends on**: Phase 1
**Requirements**: PLAY-01, PLAY-02, PLAY-03, PLAY-04, AI-01, AI-02, AI-03, CONT-02, CONT-04
**Success Criteria** (what must be TRUE):
  1. Player can create a character by choosing race, profession, and background, and see starting stats and equipment
  2. Player can explore a scene with `/look`/`/inspect`/`/scan` and see environment description, NPCs, exits, and objects
  3. Player can talk to an NPC via `/talk` and receive personality-consistent, context-aware dialogue generated by AI
  4. Player can engage in turn-based combat with `/attack`/`/cast`/`/guard`/`/flee` and see deterministic HP/MP resolution with AI-narrated outcomes
  5. AI Narrative Director generates Chinese prose (80-180 chars) per turn from adjudicated results without inventing world facts

### Phase 4: Differentiation
**Goal**: Chronicle's unique features are live -- players can branch storylines like git, view ASCII maps, browse discovered lore, use keyboard shortcuts, and the AI respects epistemic boundaries (NPCs only know what they should know)
**Depends on**: Phase 3
**Requirements**: SAVE-02, SAVE-03, SAVE-04, CLI-02, CLI-03, CLI-04, LLM-04
**Success Criteria** (what must be TRUE):
  1. Player can branch storylines (`/branch name`), switch between branches, and see a branch tree visualization
  2. Player can compare branches (`/compare`) and see human-readable differences in quests, relationships, inventory, and location
  3. Player can view an ASCII/Unicode region map via `/map` showing current location, explored areas, and points of interest
  4. Player can browse discovered lore via `/codex` with search/filter across races, factions, locations, spells, and items
  5. Retrieved context for NPC Actors is tagged with epistemic level (world_truth, npc_belief, player_knowledge) and NPCs only receive information their character would know

### Phase 5: Polish & Optimization
**Goal**: Long sessions stay fast and affordable -- background summarization compresses history, replay lets players review past turns, multi-provider LLM routing optimizes cost/quality per AI role, and token usage is visible
**Depends on**: Phase 4
**Requirements**: AI-04, LLM-01, LLM-02, LLM-03
**Success Criteria** (what must be TRUE):
  1. Background Summarizer compresses long sessions into chapter summaries and NPC memory notes without blocking gameplay
  2. Player can replay recent turns via `/replay N` by reading stored turn log (not re-generating AI output)
  3. Multiple LLM providers (OpenAI, Anthropic, Google, Qwen, DeepSeek) are supported with per-AI-role model routing configuration
  4. Player can view token usage and estimated cost per turn and per session via `/cost`
  5. Static prompt content is cached/prefixed to reduce per-turn token costs
