| Property | Value |

## Spacing Scale

Terminal spacing is measured in character cells, controlled via Ink `Box` props.

| Token | Value | Usage |
|-------|-------|-------|
| none | 0 cells | Inline elements, no gap |
| xs | paddingX={1} | Standard panel horizontal padding (established pattern) |
| sm | 1 empty `<Text> </Text>` line | Vertical separation within a section |
| md | 2 empty lines or `<Divider>` | Section breaks between panels |

Exceptions: none. All existing panels use `paddingX={1}` consistently. Phase 7 components must follow the same pattern.

**Source:** Pre-populated from codebase scan -- ScenePanel, ActionsPanel, DialoguePanel, StatusBar all use `paddingX={1}`.

## Typography

Terminal typography is controlled via Ink `<Text>` props. No pixel sizes -- terminal characters are monospace at the user's configured font size.

| Role | Text Decoration | Usage |
|------|----------------|-------|
| Body | `<Text>` (no props) | Narration lines, normal text, streamed content |
| Label | `<Text dimColor>` | Helper text, hints, inactive items, streaming indicator |
| Heading | `<Text bold>` | Section headers ("可选行动"), panel titles |
| Accent | `<Text bold color="cyan">` | NPC names, active selections, important labels |

Weight mapping:
- Regular = no `bold` prop (used for body text, narration)
- Bold = `bold` prop (used for headings, selected items, accent elements)

**Source:** Pre-populated from codebase scan -- established pattern across all 20+ existing components.

## Color

Terminal ANSI color palette. The 60/30/10 split is adapted to terminal context.

| Role | ANSI Value | Usage |
|------|------------|-------|
| Dominant (60%) | default terminal foreground (no `color` prop) | Narration text, body content, dialogue text, streamed text |
| Secondary (30%) | `dimColor` | Inactive items, helper hints, streaming indicator suffix, empty states |
| Accent (10%) | `cyan` | Selected action items, NPC name headers, active focus indicators |
| Warning | `yellow` | Skill check tags, confirmation prompts, low HP warning |
| Destructive | `red` | Critical HP (<10%), error messages |

Accent (`cyan`) reserved for:
- Selected action in ActionsPanel (`color={isSelected ? 'cyan' : undefined}`)
- NPC name in DialoguePanel (`<Text bold color="cyan">`)
- Title screen header (`<Text bold color="cyan">`)
- Combat header (`<Text bold color="cyan">`)

**New for Phase 7 -- streaming indicator:** dim ellipsis appended to streaming line. Uses `dimColor`, NOT accent. The streaming line itself renders in default foreground (same as completed narration lines).

**Source:** Pre-populated from codebase scan. Color usage is 100% consistent across all existing components.

## Streaming Visual Contract

This section defines the visual behavior specific to Phase 7's streaming output feature.

### Streaming Line Rendering (STREAM-01, STREAM-02)

| State | Visual Treatment |
|-------|-----------------|
| Streaming active, buffer has content | `<Text>{flushedText}<Text dimColor>...</Text></Text>` -- accumulated flushed text followed by dim ellipsis indicator |
| Streaming active, buffer empty (waiting) | Last flushed line visible; dim ellipsis on next line: `<Text dimColor>...</Text>` |
| Stream complete | Dim ellipsis removed. Final text becomes a normal `lines[]` entry. No visual difference from existing narration lines. |
| Stream error | `<Text>[叙事错误] {message}</Text>` -- matches existing error pattern in `handleActionExecute` |

**Streaming text is visually indistinguishable from static narration text.** The only visual difference during active streaming is the trailing dim ellipsis `...` which signals "more content incoming." When streaming completes, the ellipsis disappears and the text joins the normal `lines: string[]` array.

### ScenePanel Props Extension

```
Current:  lines: readonly string[]
New:      lines: readonly string[]
          streamingText?: string        // in-progress line (flushed buffer content)
          isStreaming?: boolean          // controls dim ellipsis visibility
```

When `isStreaming` is true and `streamingText` has content:
- Render all `lines` normally (static, completed)
- Render `streamingText` as the final visible line, followed by `<Text dimColor>...</Text>`
- When `isStreaming` becomes false, `streamingText` is cleared and its final value was already committed to `lines`

### Skip-to-End Visual Feedback (STREAM-03)

| State | Visual Treatment |
|-------|-----------------|
| Streaming, skip not pressed | Dim ellipsis `...` visible after streaming text |
| Player presses Enter/Space | Ellipsis immediately disappears. All received text renders as complete lines. No flash, no transition animation. |
| Background stream still running after skip | No visual indicator. Text appears settled. When stream completes, content is silently updated if new text arrived after skip. |
| inputMode during/after skip | Remains `processing` (ActionsPanel `isActive=false`, input disabled). Transitions to `action_select` only after stream fully completes + metadata processed. |

**No loading spinner for streaming.** The streaming text itself IS the loading indicator. An empty scene with just `<Text dimColor>...</Text>` signals "AI is thinking" before the first buffer flush arrives.

### Hint Bar During Streaming

During `inputMode === 'processing'` with active streaming, the ActionsPanel hint text changes:

| State | Hint Text |
|-------|-----------|
| Normal (action_select) | `↑↓ 选择    Enter 确认    / 输入自定义行动` (existing) |
| Streaming active | `Enter/Space 跳过动画` |
| Streaming complete, processing metadata | `处理中...` |

This replaces the existing hint line in ActionsPanel only during streaming. After `inputMode` returns to `action_select`, the original hint restores.

### NPC Dialogue Streaming (STREAM-02)

NPC dialogue streaming uses the SAME visual treatment as narration streaming:
- Inline dialogue mode: streaming text appears as `{npcName}："{streamingText}<Text dimColor>...</Text>"`
- Full dialogue mode: streaming text appears in the DialoguePanel NPC speech area with trailing dim ellipsis
- Response options do NOT appear until stream completes and metadata is extracted

**Source:** D-07 mandates identical buffer flush strategy. This extends to identical visual treatment.

## Interaction Contract

### Input State Machine During Streaming

```
action_select  --[Enter on action]--> processing (streaming starts)
processing     --[Enter/Space]------> processing (skip: animation stops, stream continues)
processing     --[stream completes]--> action_select (metadata processed, actions refreshed)
processing     --[error]-------------> action_select (error line added to ScenePanel)
```

Keys consumed during `processing` + streaming:
- **Enter**: triggers skip-to-end (D-09)
- **Space**: triggers skip-to-end (D-09)
- **All other keys**: ignored (no panel switching, no typing, no navigation)

Keys consumed during `processing` + post-skip (stream still running in background):
- **All keys**: ignored. Player waits for stream completion.

### Input Guard

`ActionsPanel.isActive` = false during processing (existing behavior).
`InputArea.isActive` = false during processing (existing behavior).
Additional: `useInput` skip handler is active ONLY when `isStreaming === true`. After skip or stream completion, the handler deactivates.

**Source:** D-09 (Enter/Space only), D-10 (inputMode stays processing until complete).

## Copywriting Contract

| Element | Copy | Language |
|---------|------|----------|
| Streaming indicator | `...` (dim) | Universal |
| Skip hint | `Enter/Space 跳过动画` | Chinese |
| Processing hint | `处理中...` | Chinese |
| Empty scene (no narration yet) | `周围一片寂静，什么也没有发生。` (existing) | Chinese |
| Stream error (narration) | `[叙事错误] {error.message}` (existing pattern) | Chinese |
| Stream error (NPC dialogue) | `[对话错误] {error.message}` | Chinese |
| NPC dialogue fallback | `{npcName}沉默了片刻，似乎在思考。` | Chinese |
| Metadata extraction fallback | (no visible copy -- falls back to generateObject silently) | N/A |

**No empty state specific to streaming.** If the stream produces no content (edge case), the existing ScenePanel empty state (`周围一片寂静，什么也没有发生。`) handles it.

**No destructive actions in this phase.** Streaming is read-only display. No confirmations needed.

**Source:** Existing error patterns from `game-screen.tsx` line 146 and line 162. New copy items are minimal and consistent with existing Chinese-first style.
