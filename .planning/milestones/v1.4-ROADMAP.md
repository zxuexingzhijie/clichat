# Milestone v1.4: AI Quality & Game Completeness

**Status:** ✅ SHIPPED 2026-04-30

## Overview

Fix critical AI architecture violations discovered in code audit, implement true multi-turn NPC dialogue, complete AI output quality improvements, add enemy loot system, and ship a publishable build. ## Milestone Summary

**Decimal Phases:** None — phases 17-21 all linear.

**Key Decisions:**

- ARCH-01: storyAct used as raw identifier (act1/act2/act3) in NPC prompt, no translation
- ARCH-02: adjudicateTalkResult wraps sentimentToDelta; NpcSentiment inferred via `NpcDialogue['sentiment']`
- DIAL-01: multi_turn uses SystemModelMessage (role:'system') as first element for uniform provider handling
- DIAL-02: dialogueHistory migrated to {role,content}[] — single history channel, historySection deleted
- DIAL-03: messagesRef (useRef) for cross-render accumulation; reset() preserves history; resetMessages() sets session boundary
- AI-05: callGenerateObject replaces callGenerateText in generateNarration; schema rejection → catch fallback
- AI-06: classifyIntent uses callGenerateObject with role 'retrieval-planner'; model? removed from ClassifyIntentOptions
- AI-07: runSummarizerLoop checks signal.aborted at 3 points; SIGINT handler stored as named const for process.off
- GAME-01: EnemySchema loot→loot_table; SceneStateSchema droppedItems:string[] default []
- SAVE-01: SaveDataV6Schema extends V5; readSaveData() upgraded; branch-diff SaveDataCompare extends to V4|V5|V6
- DIST-01: version→1.4.0; author=Makoto; repository.url=git+https:// canonical format; npm publish --dry-run passes cleanly
- UAT-01: Live UAT written as non-blocking human checklist; automation gate (bun test + tsc + dry-run) is the pass condition

**Issues Resolved:**

- Critical architecture violation: LLM sentiment was directly mutating relationship state, bypassing Rules Engine
- NPC actor was discarding narrativeContext with `void` — no story act injected into NPC prompts
- ai-caller.ts single-turn only: now supports full messages[] multi-turn API
- generateNarration used manual slice for length control: replaced with Zod schema enforcement
- intent classification tokens were invisible to cost tracker
- Summarizer had no graceful shutdown path — Ctrl-C caused unhandled rejection

**Issues Deferred:**

- Live API UAT (3 tests: :cost, :replay, summarizer) — requires live API key; non-blocking human checklist
- Phase 05/12 VERIFICATION human_needed items — acknowledged at v1.3 close
- CJK text rendering full audit — partial mitigation via string-width sufficient

**Technical Debt Incurred:**

- 21-UAT-CHECKLIST.md pending items require manual execution with live API key before first real publish
