**Status:** ✅ SHIPPED 2026-04-29

## Overview

Introduced a full narrative arc system for Chronicle CLI: a persistent story state that evolves based on player choices, an 8-stage main quest with 3 branching endings, act-aware AI prompt injection, NPC trust-gated knowledge disclosure, and deterministic location description overrides driven by world flags. All systems are backward-compatible with V4 saves via the new SaveDataV5 migration chain.

## Milestone Summary

**Key Accomplishments:**

1. **NarrativeState store + SaveDataV5** — `NarrativeStateSchema` with `currentAct`/`atmosphereTags`/`worldFlags`/`playerKnowledgeLevel`; V5 migration chain; `narrative-transitions.yaml` maps 8 quest stage IDs to act/flag transitions; watcher subscribes to `quest_stage_advanced` at boot
2. **Act-aware narrative prompts** — `buildNarrativeSystemPrompt` injects act tone + atmosphere tags; `buildNpcSystemPrompt` supports trust-gated knowledge disclosure at 3 tiers (< 5 / 5–8 / > 8); wired into scene-manager + dialogue-manager call sites
3. **quest_main_01 6-stage arc** — `ConditionalNextStageSchema` + `resolveNextStage`; main quest expanded from 3 to 8 stages (5 linear + 3 branching endings: justice/harmony/shadow); flag-driven routing at `stage_allies_decision`
4. **NPC knowledge_profile** — `NpcKnowledgeProfileSchema` in Zod; `knowledge_profile` populated for 7 story NPCs (王德/老陈/林婆婆/月华/瞎子孙/阿鬼/陈铁柱); dialogue trust injection via `personalTrust → 0-10` scale; route-lock flags set on `endDialogue` at alliance stage
5. **Location description_overrides** — `description_overrides: Record<string, string>` on `LocationSchema`; `selectLocationDescription` with `OVERRIDE_PRIORITY` list; `handleLook` and `loadScene` bypass LLM when a worldFlag matches an override

**Key Decisions:**

- `SaveDataV4` used as base type for `compareBranches` via `SaveDataCompare` union (V4|V5) — branch-diff remains compatible with both save versions
- `narrativeStore` wired into `createGameContext` so watcher starts at app boot (not lazy)
- Trust scale: `personalTrust=0` maps to `trustLevel=5` (neutral), not < 5 — formula `(personalTrust + 100) / 20` is semantically correct; test adjusted
- `quest` field in `createDialogueManager stores` is optional to avoid breaking existing test fixtures
- `OVERRIDE_PRIORITY` list is hardcoded in scene-manager code (not schema) — avoids data migration when priority changes
- `selectLocationDescription` exported as pure function for direct unit testability; `handleLook` uses `stores.scene.getState().sceneId` (not closure variable) to avoid stale ref bug

**Issues Resolved:**

- NarrativeStore mock shape in test files missing `worldFlags` + `playerKnowledgeLevel` — auto-fixed during P01
- `SaveDataV3` references in 5 UI files after V5 introduction — auto-fixed during P01
- `generateNpcDialogue` signature didn't accept 7th `trustLevel` param — auto-fixed during P04

**Issues Deferred:**

- Phase 05/12 VERIFICATION [human_needed] — live game session UAT deferred to v1.4
- Phase 05 HUMAN-UAT [deferred] — manual game flow testing deferred
- Live API session validation (/cost, /replay, summarizer) — carried from v1.1

**Technical Debt:**

- `ai-caller.ts` interface only supports `system + prompt` (single-turn); multi-turn dialogue context is serialized to strings. True multi-turn conversation would require `messages[]` parameter — deferred to v1.4.
- Character creation guard NPC dialogue (4 rounds) uses independent LLM calls without accumulated context — same root cause as above.
