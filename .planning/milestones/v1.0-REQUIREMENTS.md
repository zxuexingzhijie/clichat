# Requirements Archive: Chronicle v1.0

**Archived:** 2026-04-22
**Core Value:** The player must feel they are in a persistent, consistent world that remembers them — not a chatbot that reinvents the universe every turn.
**Status:** ✅ All 32 v1 requirements shipped

---

## v1 Requirements (All Complete)

### Core Engine

- [x] **CORE-01**: Player can enter structured commands (`:look`, `:go north`, `:talk NPC`, `:attack`) and receive deterministic parsing into game actions — **Phase 1** ✓ Validated
- [x] **CORE-02**: Player can enter free-form natural language and have it classified into structured intent (action type, targets, modifiers) via fast LLM — **Phase 1** ✓ Validated
- [x] **CORE-03**: Rules Engine adjudicates all mechanical outcomes (success/failure, damage, resource changes, relationship deltas) deterministically based on character stats, modifiers, and randomization — with zero LLM involvement — **Phase 1** ✓ Validated
- [x] **CORE-04**: All game state exists as a single serializable JSON tree with immutable updates, enabling complete snapshot/restore at any point — **Phase 1** ✓ Validated (V4 schema with chained migrations)

### AI System

- [x] **AI-01**: Narrative Director generates 80–180 character Chinese prose per turn from adjudicated results + retrieved context, without inventing world facts or overriding game state — **Phase 2** ✓ Validated
- [x] **AI-02**: NPC Actor generates per-character dialogue informed by NPC identity, goals, current emotions, episodic memory, and scene context — each NPC only knows what they would know — **Phase 2** ✓ Validated
- [x] **AI-03**: Retrieval Planner decides which codex entries, NPC memories, and quest states to fetch for each turn, outputting a structured retrieval plan (not a prompt dump) — **Phase 2** ✓ Validated
- [x] **AI-04**: Background Summarizer compresses long sessions into chapter summaries and NPC memory notes without blocking interactive gameplay — **Phase 5** ✓ Validated (code; live session UAT deferred)

### Gameplay

- [x] **PLAY-01**: Player can create a character by choosing race, profession, and background, with corresponding base stats and starting equipment — **Phase 2** ✓ Validated
- [x] **PLAY-02**: Player can explore scenes with `:look`, `:inspect`, `:scan` — seeing environment description, NPCs present, exits, and interactable objects — **Phase 2** ✓ Validated
- [x] **PLAY-03**: Player can engage in NPC dialogue via `:talk` and `:ask` — NPCs respond with personality, memory of past interactions, and emotional state — **Phase 2** ✓ Validated
- [x] **PLAY-04**: Player can engage in turn-based combat via `:attack`, `:cast`, `:guard`, `:flee` — with deterministic resolution, HP/MP tracking, and narrated outcomes — **Phase 2** ✓ Validated

### World Data

- [x] **WORLD-01**: World Codex exists as tagged YAML files (races, locations, factions, spells, items, history events) with schema validation and required fields — **Phase 1** ✓ Validated
- [x] **WORLD-02**: Per-NPC episodic memory stored as structured JSON (three-layer: recent/salient/archive), tagged with participants, locations, quest IDs, and emotional valence — **Phase 3** ✓ Validated
- [x] **WORLD-03**: Player can accept, track progress on, and complete quests — with a `:journal` command showing active/completed/failed quests — **Phase 3** ✓ Validated
- [x] **WORLD-04**: Relationship system tracks player reputation with individual NPCs and factions, influencing NPC dialogue and quest availability — **Phase 3** ✓ Validated

### Save & Branch

- [x] **SAVE-01**: Player can quick save, named save (`:save "before the gate"`), and load any save — full game state serialized/restored — **Phase 3** ✓ Validated
- [x] **SAVE-02**: Player can branch storylines (`:branch name`) to explore alternative decisions, with branch tree visualization — **Phase 4** ✓ Validated
- [x] **SAVE-03**: Player can compare branches (`:compare main..branch/name`) seeing state differences (quests, relationships, inventory, location) as human-readable summary — **Phase 4** ✓ Validated
- [x] **SAVE-04**: Player can replay recent turns (`:replay N`) by reading stored turn log (not re-generating AI output) — **Phase 4/5** ✓ Validated (code; live session UAT deferred)

### CLI UX

- [x] **CLI-01**: Terminal displays four-panel layout: scene panel (narration + NPCs + exits), status bar (HP/MP/reputation/quest), suggested actions, and input area — responsive to terminal resize — **Phase 1** ✓ Validated
- [x] **CLI-02**: ASCII/Unicode region map accessible via `:map` showing current location, explored areas, exits, and points of interest — **Phase 4** ✓ Validated
- [x] **CLI-03**: World codex browser via `:codex` allowing search/browse of discovered lore entries (races, factions, locations, spells, items) — **Phase 4** ✓ Validated
- [x] **CLI-04**: Keyboard shortcuts: Tab completion for commands/NPC names, arrow key history, Ctrl-R history search, `?` quick help — **Phase 4** ✓ Validated

### LLM Infrastructure

- [x] **LLM-01**: Multi-provider abstraction layer supporting OpenAI, Anthropic, Google, Alibaba Qwen, and DeepSeek — with per-AI-role model routing configuration via YAML — **Phase 5** ✓ Validated
- [x] **LLM-02**: Token usage and estimated cost tracked per turn and per session, visible via `:cost` command — **Phase 5** ✓ Validated (code; live session UAT deferred)
- [x] **LLM-03**: Static prompt content (world rules, narrative style, character skeletons) cached/prefixed to reduce per-turn token costs — **Phase 5** ✓ Validated
- [x] **LLM-04**: Retrieved context tagged with epistemic level (world_truth, npc_belief, player_knowledge) — NPC Actors only receive information their character would know — **Phase 4** ✓ Validated

### Content

- [x] **CONT-01**: First region world pack: one region with 9 locations (towns, dungeons, wilderness), 4 factions, regional lore, danger levels — **Phase 3** ✓ Validated
- [x] **CONT-02**: Character system: 3-4 races with traits, 3-4 professions with abilities, 5-6 base stats, starting equipment per profession — **Phase 2** ✓ Validated
- [x] **CONT-03**: Quest content: one main quest skeleton and reusable side quest templates with region/NPC/constraint parameters — **Phase 3** ✓ Validated
- [x] **CONT-04**: NPC content: 15+ named NPCs with backstories, goals, relationships, initial memory seeds, and personality tags — **Phase 2** ✓ Validated
