## ARCH — AI Architecture Integrity

- [x] **ARCH-01**: NPC Actor receives `narrativeContext` (currentAct + atmosphereTags) and uses it in system prompt generation — NPC dialogue tone must reflect current story act
  - **Outcome:** buildNpcSystemPrompt extended with optional 3rd param; npc-actor.ts void bug fixed; Phase 17 P01 ✓
- [x] **ARCH-02**: NPC `sentiment` output is NOT applied directly as relationship delta; dialogue outcome is passed to Rules Engine `adjudicateTalkResult`, which determines the relationship change
  - **Outcome:** adjudicateTalkResult added to rules-engine.ts; both dialogue-manager call sites routed through it; Phase 17 P02 ✓

## DIAL — Multi-Turn Dialogue

- [x] **DIAL-01**: `ai-caller.ts` accepts optional `messages: Array<{role:'user'|'assistant', content:string}>` parameter; when present, sends multi-turn conversation structure to LLM API instead of single system+prompt pair
  - **Outcome:** multi_turn MessageMode branch added; SystemModelMessage forwarding; Phase 18 P01 ✓
- [x] **DIAL-02**: `DialogueManager` maintains `dialogueHistory` as proper `{role, content}` array per dialogue session; history passed to LLM via `messages[]` on each NPC turn
  - **Outcome:** Atomic {speaker,text}→{role,content} migration across 4 consumers; historySection deleted; Phase 18 P02 ✓
- [x] **DIAL-03**: Character creation guard dialogue (4 rounds) uses accumulated `messages[]` context — each round builds on prior exchanges without re-serializing to plain text
  - **Outcome:** messagesRef (useRef) in useNpcDialogue; guard 4-round wiring; Phase 18 P03 ✓

## AI — AI Output Quality

- [x] **AI-05**: `generateNarration` in `narrative-director.ts` uses `callGenerateObject` with `NarrationSchema` (`text: z.string().max(300)`, `mood: z.enum(...)`, `suggestedActions?: z.array(z.string()).max(3)`) instead of `callGenerateText` + manual slice
  - **Outcome:** NarrationOutputSchema created; callGenerateObject replaces callGenerateText; Phase 19 P01 ✓
- [x] **AI-06**: `intent-classifier.ts` routes through `callGenerateObject` in `ai-caller.ts`; intent classification token usage is recorded via `recordUsage()` and counted in `:cost` output
  - **Outcome:** classifyIntent uses callGenerateObject with role 'retrieval-planner'; intent tokens visible in :cost; Phase 19 P02 ✓
- [x] **AI-07**: `runSummarizerLoop` accepts `AbortSignal`; loop exits cleanly when signal is aborted; game shutdown calls abort; summarizer errors are logged (not silently swallowed)
  - **Outcome:** AbortSignal at 3 check points; SIGINT handler wired in app.tsx; Phase 19 P03 ✓
