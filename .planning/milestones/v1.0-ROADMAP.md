# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-04-22
**Phases:** 1–5
**Total Plans:** 37

## Overview

Chronicle v1.0 delivers a fully playable AI-driven CLI text RPG. A deterministic Rules Engine
adjudicates all outcomes while AI generates Chinese narration and NPC dialogue. Players explore
a Classic Fantasy region, fight enemies, talk to memory-enabled NPCs, manage quests and
relationships, branch storylines like git, view ASCII maps, browse discovered lore, and track
LLM token costs — all from a four-panel terminal UI built with React + Ink.

## Milestone Summary

**Key Decisions:**

- TypeScript + Bun runtime — matches Claude Code reference; fast cold start; native TS execution
- Tool-based RAG over vector DB — human-readable YAML codex, no embedding infrastructure
- Rules Engine has NO LLM access — deterministic adjudication; AI cannot override game state
- Three epistemic layers (world_truth / npc_belief / player_knowledge) — NPCs are not omniscient
- Custom createStore pattern (~35 lines) — no Zustand/Redux; game logic is UI-independent
- Immer for complex nested state — selective use, not everywhere
- All 6 AI roles default to gemini-2.0-flash placeholder — real provider routing via YAML config in Phase 5
- NpcMemoryRecord three-layer schema (recent/salient/archive) — enables lossless summarization
- SaveData versioned migrations (V1→V2→V3→V4) — forward-compatible save files
- Inline dialogue mode — NPC speech appended as narration lines; no layout change required
- CombatActionResult discriminated union (ok|error) — callers narrow before accessing results
- QUEST_GOAL_KEYWORDS excludes 'protect' — avoids false full-mode triggers on routine guards
- CostSessionState is ephemeral — resets on state_restored; prevents cross-session cost bleed
- Version conflict in applyNpcMemoryCompression fails task, does NOT re-queue — preserves NPC memories

**Issues Resolved:**

- TypeScript errors from AI SDK v5 LanguageModelV3 cast, maxOutputTokens, usage nullability resolved
- Phase 5 WR-01–05 code review fixes applied atomically

**Issues Deferred:**

- 3 Phase 5 human UAT items (live /cost, /replay panel, background summarizer) — behavioral checks requiring live API session; carry to v1.1 first-play
- chapter_summary and turn_log_compress store results in module-level stub arrays — display not wired to game-screen (Phase 6 candidate)
- CJK text rendering with Ink 7 Box components needs live terminal testing

**Technical Debt:**

- QUEST_GOAL_KEYWORDS is a static list — should be configurable per world pack
- Reputation events fire during game load (WR-01 from Phase 3) — double-count risk on load
- endDialogue uses accumulated relationshipValue including initial_disposition as delta — scale mismatch risk noted
- applyRetention promotes oldest memory, not highest importance — acceptable for MVP
