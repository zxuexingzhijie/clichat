### WR-01: `resolveCharacter` can return empty string `raceId` — `buildCharacter` silently uses it as display name

**Files modified:** `src/ui/screens/narrative-creation-screen.tsx`
**Commit:** `cc0321a`
**Applied fix:** In `handleNameSubmitted`, added fallback to `buildCharacter` call: `raceId: resolved.raceId || 'race_human'` and `professionId: resolved.professionId || dialogueConfig.archetypePriority.profession[0] || 'prof_adventurer'`. ### WR-02: `skipToEnd` in `use-streaming-text` fires `setStreamingText` twice

**Files modified:** `src/ui/hooks/use-streaming-text.ts`
**Commit:** `3935968`
**Applied fix:** In the `finally` block, gated `setStreamingText(fullTextRef.current)` behind `if (!skippedRef.current)`. When `skipToEnd` is called mid-stream it already sets `streamingText` to the accumulated `fullTextRef.current`; the `finally` block now skips the redundant second call, eliminating the double render / flash-to-complete-text UX artifact.

### WR-03: `chapterSummaries` passed to `PanelRouter` is a non-reactive read

**Files modified:** `src/ui/screens/game-screen.tsx`
**Commit:** `3e7ca28`
**Applied fix:** Replaced the inline `chapterSummaries={getRecentChapterSummaries()}` prop with React state: `const [chapterSummaries, setChapterSummaries] = useState(() => getRecentChapterSummaries())`. Added a `useEffect` that subscribes to `eventBus.on('summarizer_task_completed', ...)`, and on payloads where `p.type === 'chapter_summary'` calls `setChapterSummaries(getRecentChapterSummaries().slice())` to produce a new array reference and trigger a re-render. Subscription is cleaned up on unmount via the returned `() => eventBus.off(...)`.
