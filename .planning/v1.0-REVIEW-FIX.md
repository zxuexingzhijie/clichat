### CR-02: questSystem created before codex loads — misses all auto-accept triggers

**Files modified:** `src/engine/quest-system.ts`, `src/app.tsx`
**Commit:** 3166ba1

### WR-01: `quest_stage_advanced` / `quest_completed` bus events not emitted on save restore

**Files modified:** `src/state/quest-store.ts`, `src/state/serializer.ts`, `src/context/game-context.ts`
**Commit:** 167cfb9
**Applied fix:** Added `QuestStore` type (extending `Store<QuestState>` with `restoreState`) and updated `createQuestStore` to use an `isRestoring` flag (matching the `relationStore` pattern). `restoreState` sets the flag, calls `setState`, then clears it, suppressing all bus events during bulk restore. Updated `serializer.ts` to accept `QuestStore` instead of `Store<QuestState>` and call `restoreState` instead of `setState` for quest data. Also removed the redundant `stores.quest.setState(draft => { draft.eventLog = data.questEventLog; })` call (IN-02) since `restoreState` already restores `eventLog` as part of the full quest state. Updated `game-context.ts` to use `QuestStore` type.

---

### WR-03: `quest-system.ts` auto-accept logic has inverted guard for already-active quests

**Files modified:** `src/engine/quest-system.ts`
**Commit:** 8571ba0
**Applied fix:** Changed the auto-accept guard from `if (progress && progress.status !== 'failed') continue` to `if (progress && (progress.status === 'active' || progress.status === 'completed')) continue`. This allows `unknown`-status quests to be re-accepted and prevents failed quests from being re-accepted on every `dialogue_ended` event.

---

### WR-04: `save-migrator` `migrateV3ToV4` is a no-op — v4 schema never validated

**Files modified:** `src/state/serializer.ts`
**Commit:** 2218f09
**Applied fix:** Updated `restore()` to attempt `SaveDataV4Schema.safeParse` first, then fall back to `SaveDataV3Schema.safeParse` if V4 fails. This ensures saves written with `version: 3` that were not migrated through the chain (e.g., written before the V4 schema was introduced) can still be loaded. Error reporting still uses the V4 parse error for diagnostics.
